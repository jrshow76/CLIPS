## 0. 문서 개요

Planner의 86개 화면 중 비즈니스 가치·구현 난이도·시연 효과를 고려해 **우선순위 12개** 화면을 선정해 ASCII/Markdown 박스 와이어프레임으로 설계한다. 각 화면은 다음 요소를 포함한다.

- 화면 ID, 메뉴 ID, 권한, 핵심 사용자, 디바이스
- ASCII 레이아웃 다이어그램
- 핵심 UI 요소 표
- 사용자 시나리오 3개 이상
- 반응형 변형 (PC / Tablet / Mobile)
- 예외 상태 (빈상태 / 로딩 / 오류 / 권한없음)

### 1.2 핵심 UI 요소

| 요소 | 컴포넌트 | 데이터 |
|---|---|---|
| KPI 카드(4) | StatCard | 오늘대출/반납대기/연체/좌석점유율 |

| 활동 피드 | NotificationCenter | 최근 20건 |

### 1.3 사용자 시나리오

1. **개관 직후 일일 점검**: 사서가 출근 후 대시보드를 열어 오늘대출·반납대기·연체 KPI를 1초 안에 파악한다.
2. **결재 처리**: 처리대기 표에서 "발주 승인 3건"의 "바로가기"를 눌러 발주 관리 화면으로 이동한다.
3. **이상 감지**: 좌석 점유율 KPI가 평년대비 빨간색이면 클릭해 좌석 현황으로 드릴다운한다.

### 1.4 반응형

- **태블릿 1024**: KPI 4→2열 그리드, 차트 1열, 표 그대로
- **모바일 미지원** (사서 대시보드는 PC 우선)

### 1.5 예외 상태

| 상태 | 표시 |
|---|---|
| 로딩 | 카드/차트 Skeleton |
| 빈 처리대기 | EmptyState "처리 대기 작업이 없습니다 ✨" |
| 데이터 오류 | 카드별 인라인 에러 + 재시도 버튼 |

### 2.2 핵심 UI 요소

| 요소 | 컴포넌트 | 동작 |
|---|---|---|
| 임시저장 리스트 | List + Badge | 동시 여러 서지 작업 |
| MARC 그리드 | MarcEditor | 인라인 편집, 단축키 |
| 필드 라벨 자동완성 | Combobox | 태그 번호 입력 시 자동완성 |
| 권위 일치 표시 | Badge `[✓ 권위]` | 권위레코드 매칭 |
| 검증 패널 | 오류·경고·정보 분류 | 인라인 이동 가능 |
| Z39.50 import | Modal | 외부 검색 결과 → MARC 덮어쓰기/병합 |

### 2.3 사용자 시나리오

1. **신규 편목**: 사서가 ISBN으로 Z39.50 검색 → 결과 선택 → MARC import → 보정 → 저장.
2. **임시저장 복귀**: 점심 후 임시저장함에서 작업중이던 서지를 클릭해 정확히 같은 상태 복원.
3. **검증 통과**: 검증 패널의 "020 ISBN 형식 오류" 클릭 → 해당 필드 포커스 이동 → 수정 → 검증 재실행.
4. **권위 일치**: 650 주제어 입력 시 권위레코드 자동 매칭, 일치하면 `[✓ 권위]` 뱃지 자동 부여.

### 2.4 반응형

- 1280 미만: 좌·우 패널을 Drawer로 토글
- 1024 미만: "이 화면은 데스크탑 환경을 권장합니다" 안내

### 2.5 예외 상태

| 상태 | 표시 |
|---|---|
| 로딩 | MARC 그리드 Skeleton |
| 신규(빈 서지) | Leader 자동 채움 + 008 가이드 |
| 저장 실패 | Toast danger + 임시저장 자동 |
| 권한없음 | 읽기전용 모드 + 안내 배너 |
