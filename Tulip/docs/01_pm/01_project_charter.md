## 1. 프로젝트 개요

### 1.1 배경
국내 도서관 시장은 학교·대학·공공도서관별로 분절된 레거시 솔루션(상용 패키지·자체 구축)이 다수 운영 중이며, KORMARC·KOLIS-NET·DLS·KERIS 등 국가 표준 연계와 RFID 하드웨어 통합 운영의 복잡도가 높다. SaaS 기반 멀티테넌트 통합 솔루션으로 모든 유형의 도서관을 단일 플랫폼에서 운영 가능하게 함으로써, 도입·운영 비용을 절감하고 표준 준수와 데이터 상호운용성을 확보한다.

### 1.2 프로젝트 정의
- **유형**: SaaS형 멀티테넌트 도서관통합관리솔루션 (신규 구축, 마이그레이션 없음)
- **타겟**: 학교도서관 / 대학·전문도서관 / 공공도서관(단일관·다관통합) 전 유형
- **업무 도메인 6종**: 수서, 목록, 열람, 장서관리, 출입관리, 시설관리
- **표준 준수**: KORMARC(필수), MARC21, Z39.50, KOLIS-NET, DLS, KERIS
- **하드웨어**: RFID(SIP2/NCIP), 출입게이트·도난방지(EAS), 바코드, SW-only 모드 동시 지원

## 2. 비전 및 목표 (3년)

### 2.1 비전
> "모든 유형의 도서관이 단일 플랫폼에서 표준에 부합하는 업무를 수행할 수 있도록 한다."

### 2.2 3년 로드맵 목표

| 시점 | 핵심 목표 | 주요 산출물 |
|---|---|---|
| **Y1 (2026~2027)** | 6개 도메인 MVP 출시, 학교·중소공공 5개 기관 베타 운영 | Phase 1~5 완료, 운영형 SaaS 1.0 |
| **Y2 (2027~2028)** | 대학·다관통합 지원, 통계·확장 모듈, 유료 기관 30개 확보 | Phase 6 완료, 대시보드·통계 고도화 |
| **Y3 (2028~2029)** | AI 기반 추천·자동분류, 외부 컨소시엄(KOLIS-NET) 정식 인증, 100개 기관 확보 | 확장 도메인 출시, 표준 인증 획득 |

## 5. 프로젝트 범위

### 5.1 In Scope (포함)

#### 5.1.1 업무 도메인 (6종)
- **수서**: 발주·검수·구입·기증·예산관리·연속간행물 발주
- **목록**: KORMARC/MARC21 편목, 사본·소장정보, 권위 통제, Z39.50 복사편목
- **열람**: 대출·반납·예약·연체, 회원관리, OPAC, 모바일 셀프서비스
- **장서관리**: 점검(인벤토리), 제적, 이관, 라벨 출력, 분류·청구기호 관리
- **출입관리**: 게이트 연동, 출입로그, 좌석배정(열람실), 도난방지(EAS)
- **시설관리**: 회의실·세미나실 예약, 시설 점검, 분실물

#### 5.1.2 공통 기반
- 멀티테넌트 아키텍처 (Row-level 격리 + 옵션별 Schema 분리)
- 통합 인증 (OAuth2 / JWT / SSO·교육행정정보시스템 연계)
- 권한관리 (RBAC + ABAC)
- 외부 표준 연계 (KOLIS-NET, DLS, KERIS, Z39.50)
- 하드웨어 게이트웨이 (SIP2/NCIP, EAS, RFID 리더)
- 운영 대시보드·통계

### 5.2 Out of Scope (제외)

| 항목 | 사유 | 향후 검토 |
|---|---|---|
| 레거시 데이터 마이그레이션 | 신규 구축 프로젝트 | 별도 SI 프로젝트 |
| 회계·예산 ERP 연동 | 1차 범위 외 | Y2 검토 |
| 도서 추천 AI/ML | Y3 로드맵 | Phase 7+ |
| 전자책 DRM 자체 개발 | 외부 솔루션 사용 | 위탁 운영 |
| 도서관 홈페이지 CMS 풀세트 | OPAC 외 별도 | 외부 CMS 연동만 |
| RFID 하드웨어 자체 제조 | 표준 통신만 지원 | — |
| 오프라인 설치형(On-Premise) | SaaS 전용 | Y3 검토 |

### 5.3 가정사항
1. 모든 베타 기관은 인터넷 연결과 표준 RFID 하드웨어를 보유한다.
2. 국립중앙도서관 KOLIS-NET 표준 문서는 공개 자료로 접근 가능하다.
3. 클라우드 인프라 비용은 별도 예산으로 확보된다.
4. 6개 도메인의 업무 표준은 KOLIS·KERIS 권고안을 1차 기준으로 한다.

### 5.4 제약사항
1. **법적**: 개인정보보호법, 전자정부법, 클라우드 보안인증(CSAP) 준수.
2. **표준**: KORMARC 입출력 호환성은 KOLIS-NET 시험 통과가 전제.
3. **기술**: Java 17+, Spring Boot 3.x, PostgreSQL, Next.js, Docker로 스택 고정.
4. **일정**: Phase 1~5는 Y1 내 완료 필수.
5. **하드웨어**: 시중 유통되는 RFID·EAS 주요 벤더(3M, 비블리오테카, 국내 RFID솔루션)와 SIP2/NCIP 호환만 보장.

1. **요청 접수**: 누구나 변경요청서(CR) 작성 가능.
2. **영향도 분석**: Planner(기능)·DevLead(기술)·DBA(데이터)가 영향도 평가.
3. **승인 게이트**:
   - 일정 영향 ≤ 3일·작업 ≤ 8h: DevLead 승인.
   - 일정 영향 4~10일: PM 승인.
   - 일정 영향 > 10일 또는 범위 추가: 스폰서 승인.
4. **반영**: WBS·리스크 등록부 동시 업데이트.
