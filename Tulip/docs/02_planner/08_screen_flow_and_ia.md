---

## 1. 문서 목적

Tulip+ 도서관통합관리시스템의 **사이트맵(IA), 핵심 사용자 시나리오, 화면 목록 매트릭스**를 제시한다. 본 문서는 Designer의 와이어프레임·디자인 시스템과 FrontendSenior의 라우팅·컴포넌트 구조 설계의 기초 입력으로 활용된다.

## 2. 사용자 페르소나

| Persona | 약어 | 주 사용 영역 | 디바이스 |
|---|---|---|---|
| 시스템 관리자(플랫폼 운영자) | SA | 테넌트·플랜·모니터링 | 데스크톱 |
| 관장 | DIR | 대시보드·결재·통계 | 데스크톱·태블릿 |
| 사서(편목/수서/열람) | LIB | 전 업무 영역 | 데스크톱·핸드헬드 |
| 일반 사서·열람담당 | CIR | 대출/반납/회원 | 데스크톱·POS |
| 일반 이용자 | USR | OPAC·예약·내정보·좌석 | 모바일·데스크톱 |
| 외부 이용자(임시증) | EXT | 검색만 | 모바일·키오스크 |

## 4. 핵심 사용자 시나리오

### 4.1 시나리오 1: 희망도서 신청 → 발주 → 입고 → 대출

```mermaid
sequenceDiagram
  participant U as 이용자
  participant OPAC
  participant ACQ as 수서모듈
  participant CAT as 목록모듈
  participant COL as 장서관리
  participant CIR as 열람모듈
  U->>OPAC: 희망도서 신청(ISBN/서명)
  OPAC->>ACQ: 신청서 등록
  ACQ->>ACQ: 중복소장 체크·승인 대기
  ACQ->>U: 채택/반려 알림
  ACQ->>ACQ: 발주서 생성·납품처 전송
  ACQ->>ACQ: 납품·검수·예산 차감
  ACQ->>CAT: 서지 등록 인계(외부서지/신규)
  CAT->>CAT: KORMARC 편집·확정
  CAT->>COL: 소장 등록(채번·라벨·RFID)
  COL->>U: 입수 완료 알림
  U->>OPAC: 대출 예약(또는 카운터 방문)
  U->>CIR: 대출 처리
```

### 4.2 시나리오 2: 회원가입 → 대출 → 연체 → 완납·해제

```mermaid
sequenceDiagram
  participant U as 이용자
  participant OPAC
  participant CMN as 회원관리
  participant CIR as 열람모듈
  participant NTF as 알림
  U->>OPAC: 회원가입 신청
  CMN->>U: 본인인증·승인
  CMN->>U: 회원증(모바일QR) 발급
  U->>CIR: 자료 대출
  CIR->>CIR: 정책 검증·대출 확정
  CIR->>NTF: 만기 예정 알림(D-3)
  Note over CIR: 만기 초과
  CIR->>CIR: 연체 산정 배치·이용제한 적용
  CIR->>NTF: 연체 안내
  U->>CIR: 자료 반납
  CIR->>CIR: 연체료 산정
  U->>CIR: 연체료 납부
  CIR->>CIR: 이용제한 해제
```

### 4.3 시나리오 3: 외부서지 복사목록 → 소장 등록 → 배가

```mermaid
sequenceDiagram
  participant LIB as 사서
  participant CAT as 목록모듈
  participant Z39 as Z39.50/KOLIS-NET
  participant COL as 장서관리
  LIB->>CAT: 신간 자료 ISBN 검색
  CAT->>Z39: 외부 서지 검색
  Z39-->>CAT: 검색결과(MARC)
  LIB->>CAT: 결과 선택·import
  CAT->>CAT: KORMARC 변환·보정·저장
  CAT->>COL: 소장 등록 인계
  LIB->>COL: 등록번호 채번·라벨 인쇄·RFID 인코딩
  LIB->>COL: 배가(서가 위치 확정)
  COL->>OPAC: 검색 색인 갱신
```

### 4.4 시나리오 4: 자가대출(RFID/SIP2)

```mermaid
sequenceDiagram
  participant U as 이용자
  participant Kiosk as 자가대출기
  participant SIP as SIP2 게이트웨이
  participant CIR as 열람모듈
  participant EAS as 도난방지게이트
  U->>Kiosk: 회원증 인증
  Kiosk->>SIP: 회원 인증 요청
  SIP->>CIR: 회원 조회·정책 확인
  CIR-->>SIP: 인증결과·정책
  U->>Kiosk: 자료 안테나 인식
  Kiosk->>SIP: 대출 거래 요청
  SIP->>CIR: 대출 처리·트랜잭션
  CIR-->>SIP: 거래 결과
  Kiosk->>U: 영수증 출력·EAS 비활성화
  U->>EAS: 게이트 통과
  EAS->>CIR: 통과 이벤트(정상)
```

### 4.7 시나리오 7: 관간대차 (다관)

```mermaid
sequenceDiagram

  participant OPAC
  participant CIR_A as 신청관

  U->>CIR_A: 반납
  CIR_A->>CIR_B: 회송 통지·소장관 반환
```
