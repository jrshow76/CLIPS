## 1. 개요

### 1.1 범위
도메인 6종(수서·목록·열람·장서·출입·시설)이 공통으로 의존하는 **테넌트 관리·회원·코드마스터·권한·정책·통계** 기능을 정의한다.

### 1.2 우선순위
Phase 1 (공통기반) 단계에서 최우선 구현되어야 한다.

### 1.3 AS-IS / TO-BE
| 구분 | AS-IS (현행) | TO-BE (Tulip+) |
|---|---|---|
| 시스템 분리 | 도서관별 별도 패키지 | 단일 SaaS 멀티테넌트 |
| 회원관리 | 도서관별 회원DB | 테넌트 단위 통합회원·다관 공유 |
| 코드체계 | 패키지별 상이 | 국가표준 + 테넌트 확장 |
| 권한관리 | 사용자명/직책 매핑 | RBAC + ABAC |
| 정책관리 | 하드코딩 또는 단순 코드 | 정책엔진 기반 동적 설정 |

## 3. 비기능 요구사항

| 구분 | 요구사항 |

## 6. API 요구사항 개요

### 6.1 공통 응답 포맷

```json
{
  "success": true,
  "code": "OK",
  "message": "처리되었습니다",
  "data": { },
  "meta": {
    "page": 1,
    "size": 20,
    "total": 100,
    "tenantId": "tnt_001"
  },
  "timestamp": "2026-05-11T14:00:00+09:00",
  "traceId": "abc123"
}
```

### 6.2 멀티테넌트 전제
- 모든 API는 JWT의 `tenantId`·`branchId` 클레임을 사용하여 데이터 격리.
- 시스템관리자(플랫폼 운영자)는 명시적으로 `X-Tenant-Id` 헤더로 전환 가능.

|---|---|---|---|
| CMN-API-001 | POST | /api/v1/auth/login | 로그인 |
| CMN-API-002 | POST | /api/v1/auth/refresh | 토큰 갱신 |
| CMN-API-010 | GET | /api/v1/tenants/{id} | 테넌트 조회 |
| CMN-API-011 | GET | /api/v1/branches | 관 목록 |
