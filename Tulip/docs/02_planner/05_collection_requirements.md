## 1. 개요

### 1.1 범위
**소장 등록·등록번호 채번·장서점검·이관·재배가·교체·제적·폐기·통계·보존서가**를 다룬다.

### 1.2 AS-IS / TO-BE
| 구분 | AS-IS | TO-BE |
|---|---|---|
| 등록번호 채번 | 단일 일련번호 | 관별 분기 채번·규칙 설정 가능 |
| 장서점검 | 수기 + 일부 RFID | RFID 일괄 스캔·실시간 보고 |
| 제적 | 수기 결재 + 별도 대장 | 시스템 워크플로·이력 관리 |
| 이관·재배가 | 별도 엑셀 | 시스템 내 이력 관리 |
| 통계 | 별도 집계 | 실시간 대시보드·표준양식 |

### 1.3 핵심 업무 흐름

```mermaid
flowchart LR
  A[검수 완료 자료] --> B[등록번호 채번]
  B --> C[소장 정보 등록]
  C --> D[RFID/바코드 라벨]
  D --> E[배가]
  E --> F{이용/점검}
  F -- 이관/재배가 --> G[위치 변경]
  F -- 분실/훼손 --> H[제적 후보]
  F -- 노후/제적 --> H
  H --> I[제적 승인 워크플로]
  I --> J[폐기/기증/매각]
```

## 3. 비기능 요구사항

| 구분 | 요구사항 |
|---|---|
| 성능 | 점검 RFID 스캔 1,000건/분 처리, 결과 집계 ≤ 5분 |
| 무결성 | 등록번호 유일성 (관 기준 or 테넌트 기준 정책별) |
| 이력 | 모든 상태·위치 변경 영구 이력 |
| 정확성 | 채번 규칙 정합성 100% |
| 동시성 | 점검 진행 중 다른 작업과 충돌 방지 |

## 5. 예외 처리 정책

| 케이스 | 처리 |
|---|---|
| 채번 중복 | 트랜잭션 락·유일제약·시퀀스 활용 |
| 점검 중 대출 시도 | 정책에 따라 차단 또는 허용 |
| 제적 후 자료 발견 | 사서 승인으로 복원 가능 |
| RFID 인코딩 실패 | 재인코딩·바코드 대체 |
| 이관 중 분실 | 이관 추적·발송기관/수신기관 합의 |
| 보존서가 자료 외부대출 | 정책상 차단(권한자만 강제처리) |
