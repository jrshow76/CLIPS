## 1. 리스크 관리 기준

### 1.1 영향도 정의

| 등급 | 기준 |
|------|------|
| 상 (High) | 프로젝트 일정 2주 이상 지연 또는 핵심 기능 미구현 가능성 |
| 중 (Medium) | 프로젝트 일정 1주 내 지연 또는 품질 저하 가능성 |
| 하 (Low) | 일정 및 품질에 미미한 영향 |

### 1.2 발생 가능성 정의

| 등급 | 기준 |
|------|------|
| 상 (High) | 발생 가능성 70% 초과 |
| 중 (Medium) | 발생 가능성 30~70% |
| 하 (Low) | 발생 가능성 30% 미만 |

### 1.3 우선순위 산정 (영향도 × 발생 가능성)

| | 발생 가능성 하 | 발생 가능성 중 | 발생 가능성 상 |
|---|---|---|---|
| **영향도 상** | 중간 | **높음** | **높음** |
| **영향도 중** | 낮음 | 중간 | **높음** |
| **영향도 하** | 낮음 | 낮음 | 중간 |

### 1.4 대응 전략 정의

| 전략 | 설명 |
|------|------|
| 회피 (Avoid) | 리스크 발생 원인을 제거하거나 계획을 변경하여 리스크 자체를 없앤다 |
| 완화 (Mitigate) | 발생 가능성 또는 영향도를 줄이는 사전 조치를 취한다 |
| 전가 (Transfer) | 리스크 책임을 제3자(보험, 외주 등)에게 이전한다 |
| 수용 (Accept) | 리스크를 인지하되 별도 대응 없이 수용하고 발생 시 대처한다 |

### 1.5 리스크 상태 정의

| 상태 | 설명 |
|------|------|
| 식별 (Identified) | 리스크가 등록되었으나 아직 대응 계획 미수립 |
| 모니터링 (Monitoring) | 대응 계획 수립 완료, 주기적 모니터링 중 |
| 발생 (Occurred) | 리스크가 실제 이슈로 전환됨 |
| 종료 (Closed) | 리스크 해소 또는 해당 단계 완료 |

## 3. 이슈 관리 대장

> 이슈는 리스크가 실제로 발생한 경우 또는 사전에 식별되지 않은 문제가 발생한 경우 등록한다.

| 이슈 ID | 제목 | 심각도 | 상태 | 발생일 | 담당자 | 해결 목표일 | 비고 |
|---------|------|--------|------|--------|--------|-------------|------|
| (현재 등록된 이슈 없음) | | | | | | | |

### 이슈 심각도 정의

| 등급 | 기준 |
|------|------|
| 긴급 (Critical) | 서비스 전체 중단 또는 데이터 손실 위험 |
| 높음 (High) | 핵심 기능 사용 불가, 우회 방법 없음 |
| 보통 (Medium) | 일부 기능 오동작, 우회 방법 존재 |
| 낮음 (Low) | UI 오류, 경미한 기능 오류 |

### 이슈 상태 정의

| 상태 | 설명 |
|------|------|
| 접수 (Open) | 이슈가 등록되었으나 담당자 미배정 |
| 분석 중 (Analyzing) | 원인 분석 진행 중 |
| 처리 중 (In Progress) | 수정 또는 대응 작업 진행 중 |
| 검증 중 (Verifying) | 수정 완료 후 QA 검증 진행 중 |
| 종료 (Closed) | 이슈 해결 완료 |
