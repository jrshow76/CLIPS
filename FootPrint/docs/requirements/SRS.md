## 1. 문서 개요

### 1.1 목적

본 문서는 '발자국(Foot-Print)' 웹 애플리케이션의 개발에 필요한 요구사항을 체계적으로 정의한다.
PM으로부터 전달받은 서비스 범위를 기반으로 기능 요구사항, 비기능 요구사항, 예외 처리 정책을 명세하며,
DevLead, Designer, DBA, QA에게 전달하는 기준 문서로 활용된다.

### 1.2 범위

| 구분 | 내용 |
|------|------|
| 서비스 유형 | 웹 애플리케이션 (반응형 지원) |
| 사용자 유형 | 일반 사용자 (회원 기반) |
| 핵심 기능 | 장소 기록, 지도 시각화, 목록/검색/필터링, 카테고리 관리, 통계/분석, 사용자 인증 |
| 제외 범위 | 소셜 공유, 다중 사용자 간 팔로우/공유, 모바일 앱 (Native) |

### 1.3 참조 문서

- PM 요구사항 전달 내용 (2026-05-09)
- 화면 정의서 (`docs/requirements/screen_definition.md`)
- API 요구사항 정의서 (`docs/api/api_requirements.md`)

## 2. AS-IS / TO-BE 분석

### 2.1 AS-IS (현재 상태)

| 항목 | 현재 상태 |
|------|----------|
| 장소 기록 방식 | 노트 앱, 메모장, 카카오맵 즐겨찾기 등 파편화된 도구 사용 |
| 정보 관리 | 사진, 위치, 메모가 별개 앱에 분산 저장 |
| 시각화 | 지도 기반 통합 조회 불가 |
| 검색/필터 | 카테고리·날짜 기준 통합 검색 불가 |
| 통계 | 방문 이력 통계 제공 없음 |
| 데이터 보호 | 개인 계정 기반 보호 구조 없음 |

### 2.2 TO-BE (목표 상태)

| 항목 | 목표 상태 |
|------|----------|
| 장소 기록 방식 | 단일 웹 서비스에서 위치·날짜·사진·메모 통합 기록 |
| 정보 관리 | 카테고리 분류, 태그 기반 체계적 관리 |
| 시각화 | 인터랙티브 지도 위에 방문 장소 마커 표시, 클러스터링 |
| 검색/필터 | 키워드, 카테고리, 날짜 범위, 지역 기준 복합 필터링 |
| 통계 | 월별 방문 수, 카테고리별 분포, 지역별 분포 차트 제공 |
| 데이터 보호 | JWT 기반 인증, 사용자별 데이터 격리 |

### 2.3 개선 효과

| 기대 효과 | 설명 |
|-----------|------|
| 기록 편의성 향상 | 하나의 서비스에서 장소 정보를 통합 관리 |
| 회고 편의성 향상 | 지도·타임라인 기반 방문 이력 시각적 확인 |
| 데이터 안전성 | 회원 계정 기반 서버 저장으로 분실 위험 제거 |
| 인사이트 제공 | 통계·분석 화면으로 자신의 활동 패턴 파악 |

## 3. 시스템 범위 및 제약사항

|------|------|

| 항목 | 제약 |

## 4. 기능 요구사항

기능 ID 체계: `FR-[모듈코드]-[순번]`
- AUTH: 인증
- PLACE: 장소 기록
- MAP: 지도 시각화
- SEARCH: 검색/필터
- CATEGORY: 카테고리 관리
- STAT: 통계/분석
- MYPAGE: 마이페이지

### 4.1 사용자 인증 (AUTH)

#### FR-AUTH-01: 회원가입

| 항목 | 내용 |
|------|------|
| 기능명 | 회원가입 |
| 우선순위 | 필수 (P1) |
| 설명 | 이메일과 비밀번호로 신규 계정을 생성한다. |

**입력 항목**

| 필드 | 필수 여부 | 유효성 규칙 |
|------|----------|------------|
| 이메일 | 필수 | RFC 5322 형식, 중복 불가, 최대 255자 |
| 비밀번호 | 필수 | 8~20자, 영문+숫자+특수문자 각 1자 이상 포함 |
| 비밀번호 확인 | 필수 | 비밀번호 입력값과 일치 |
| 닉네임 | 필수 | 2~20자, 특수문자 제외 |

**처리 흐름**

1. 사용자가 회원가입 폼 입력 후 제출
2. 프론트엔드 클라이언트 사이드 유효성 검사
3. API 호출 (`POST /api/v1/auth/register`)
4. 이메일 중복 확인
5. 비밀번호 bcrypt 해싱 후 저장
6. 가입 완료 안내 후 로그인 화면으로 이동

---

#### FR-AUTH-02: 로그인

| 항목 | 내용 |
|------|------|
| 기능명 | 로그인 |
| 우선순위 | 필수 (P1) |
| 설명 | 이메일/비밀번호로 인증 후 JWT 토큰을 발급한다. |

**입력 항목**

| 필드 | 필수 여부 | 유효성 규칙 |
|------|----------|------------|
| 이메일 | 필수 | 이메일 형식 |
| 비밀번호 | 필수 | 공백 불가 |

**처리 흐름**

1. 이메일/비밀번호 입력 후 로그인 버튼 클릭
2. API 호출 (`POST /api/v1/auth/login`)
3. 인증 성공 시 Access Token(30분), Refresh Token(7일) 발급
4. Access Token: 메모리 또는 HttpOnly Cookie 저장 (DevLead와 방식 확정)
5. Refresh Token: HttpOnly Cookie 저장
6. 로그인 성공 후 지도 메인 화면으로 이동

---

#### FR-AUTH-03: 로그아웃

| 항목 | 내용 |
|------|------|
| 기능명 | 로그아웃 |
| 우선순위 | 필수 (P1) |
| 설명 | 현재 세션을 종료하고 토큰을 무효화한다. |

**처리 흐름**

1. 로그아웃 버튼 클릭
2. API 호출 (`POST /api/v1/auth/logout`)
3. 서버에서 Refresh Token 무효화
4. 클라이언트 Cookie/메모리에서 토큰 삭제
5. 로그인 화면으로 이동

---

#### FR-AUTH-04: 토큰 갱신

| 항목 | 내용 |
|------|------|
| 기능명 | Access Token 자동 갱신 |
| 우선순위 | 필수 (P1) |
| 설명 | Access Token 만료 시 Refresh Token으로 자동 재발급한다. |

**처리 흐름**

1. API 호출 시 401 응답 수신
2. Refresh Token으로 `POST /api/v1/auth/refresh` 호출
3. 신규 Access Token 발급 및 저장
4. 원래 API 요청 재시도
5. Refresh Token 만료 시 로그인 화면으로 강제 이동

---

#### FR-AUTH-05: 회원 탈퇴

| 항목 | 내용 |
|------|------|
| 기능명 | 회원 탈퇴 |
| 우선순위 | 선택 (P2) |
| 설명 | 계정과 연결된 모든 데이터를 삭제한다. |

**처리 흐름**

1. 마이페이지 > 회원 탈퇴 메뉴 접근
2. 비밀번호 재확인
3. 탈퇴 의사 최종 확인 다이얼로그 표시
4. API 호출 (`DELETE /api/v1/auth/withdraw`)
5. 장소 기록, 사진, 카테고리 등 연관 데이터 소프트 딜리트
6. 로그인 화면으로 이동

### 4.2 장소 기록 (PLACE)

#### FR-PLACE-01: 장소 등록

| 항목 | 내용 |
|------|------|
| 기능명 | 장소 등록 |
| 우선순위 | 필수 (P1) |
| 설명 | 방문한 장소의 정보를 신규 등록한다. |

**입력 항목**

| 필드 | 필수 여부 | 유효성 규칙 |
|------|----------|------------|
| 장소명 | 필수 | 1~100자 |
| 카테고리 | 필수 | 기존 카테고리 중 1개 이상 선택 |
| 방문일 | 필수 | 과거 날짜 또는 오늘 (미래 날짜 불가) |
| 위도 (latitude) | 필수 | -90.0 ~ 90.0 범위 소수점 최대 8자리 |
| 경도 (longitude) | 필수 | -180.0 ~ 180.0 범위 소수점 최대 8자리 |
| 주소 | 선택 | 최대 255자 (지도 검색 시 자동 입력) |
| 메모 | 선택 | 최대 2,000자 |
| 평점 | 선택 | 1~5 (정수) |
| 사진 | 선택 | JPG/PNG/WebP, 최대 5장, 장당 최대 10MB |
| 태그 | 선택 | 각 태그 최대 20자, 최대 10개 |

**처리 흐름**

1. 지도에서 위치 핀 설정 또는 주소 검색으로 좌표 입력
2. 장소 등록 폼 작성
3. 사진 업로드 (선택)
4. API 호출 (`POST /api/v1/places`)
5. 등록 완료 후 장소 상세 화면으로 이동

## 5. 비기능 요구사항

### 5.1 성능 요구사항

| 항목 | 기준 |
|------|------|
| API 응답 시간 | 일반 조회 API: 300ms 이하 (p95 기준) |
| 지도 초기 로딩 | 마커 500개 이하: 1초 이내 |
| 파일 업로드 | 이미지 1장(10MB): 5초 이내 |
| 목록 조회 | 10,000건 이상 데이터에서 20건 페이지 조회: 500ms 이하 |
| 동시 사용자 | 100 동시 접속 사용자 기준 정상 응답 |

### 5.2 보안 요구사항

| 항목 | 기준 |
|------|------|
| 인증 | JWT Bearer Token (Access Token 30분, Refresh Token 7일) |
| 데이터 격리 | 사용자는 본인 데이터만 접근 가능 (타 사용자 데이터 조회 불가) |
| 비밀번호 저장 | bcrypt 해싱 (cost factor 10 이상) |
| HTTPS | 전 구간 HTTPS 적용 |
| 파일 업로드 보안 | 확장자 화이트리스트, MIME 타입 검증, 파일명 서버 재생성 |
| XSS 방지 | 사용자 입력값 서버/클라이언트 이중 이스케이핑 |
| SQL Injection 방지 | PreparedStatement / ORM 파라미터 바인딩 필수 |
| CORS | 허용 Origin 명시적 설정 |

### 5.3 가용성 요구사항

| 항목 | 기준 |
|------|------|
| 서비스 가용성 | 99% 이상 (월 기준 다운타임 7.2시간 이하) |
| 배포 방식 | Blue-Green 또는 Rolling 배포 (무중단 배포) |

### 5.4 유지보수성 요구사항

## 6. 예외 처리 정책

### 6.1 공통 HTTP 오류 코드 정책

| HTTP Status | 오류 코드 | 오류 메시지 | 설명 |
|-------------|----------|------------|------|
| 400 | INVALID_INPUT | 입력값이 올바르지 않습니다. | 요청 파라미터 유효성 실패 |
| 401 | UNAUTHORIZED | 로그인이 필요합니다. | 미인증 요청 |
| 401 | TOKEN_EXPIRED | 인증이 만료되었습니다. 다시 로그인해 주세요. | Access Token 만료 |
| 403 | FORBIDDEN | 접근 권한이 없습니다. | 타 사용자 데이터 접근 시도 |
| 404 | NOT_FOUND | 요청한 데이터를 찾을 수 없습니다. | 존재하지 않는 리소스 |
| 409 | DUPLICATE | 이미 존재하는 데이터입니다. | 중복 데이터 등록 시도 |
| 413 | FILE_TOO_LARGE | 파일 크기가 허용 범위를 초과했습니다. (최대 10MB) | 파일 크기 초과 |
| 415 | UNSUPPORTED_FILE_TYPE | 지원하지 않는 파일 형식입니다. (JPG, PNG, WebP만 허용) | 허용되지 않는 파일 타입 |
| 429 | TOO_MANY_REQUESTS | 요청이 너무 많습니다. 잠시 후 다시 시도해 주세요. | Rate Limit 초과 |
| 500 | SERVER_ERROR | 서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요. | 서버 내부 오류 |

### 6.2 기능별 오류 처리

#### 인증 (AUTH)

| 상황 | 오류 코드 | 표시 메시지 | 처리 방법 |
|------|----------|------------|----------|
| 이메일 중복 | EMAIL_DUPLICATED | 이미 사용 중인 이메일입니다. | 회원가입 폼 이메일 필드 오류 표시 |
| 이메일 형식 오류 | INVALID_EMAIL | 올바른 이메일 형식을 입력해 주세요. | 클라이언트 측 즉시 검증 |
| 비밀번호 불일치 (확인) | PASSWORD_MISMATCH | 비밀번호가 일치하지 않습니다. | 클라이언트 측 즉시 검증 |
| 비밀번호 강도 미달 | WEAK_PASSWORD | 비밀번호는 8~20자, 영문·숫자·특수문자를 포함해야 합니다. | 클라이언트 측 즉시 검증 |
| 이메일/비밀번호 불일치 | AUTH_FAILED | 이메일 또는 비밀번호가 올바르지 않습니다. | 로그인 실패 메시지 표시 (계정 존재 여부 노출 금지) |
| Refresh Token 만료 | REFRESH_TOKEN_EXPIRED | 세션이 만료되었습니다. 다시 로그인해 주세요. | 로그인 화면 강제 이동 |

#### 장소 기록 (PLACE)

| 상황 | 오류 코드 | 표시 메시지 | 처리 방법 |
|------|----------|------------|----------|
| 미래 날짜 입력 | FUTURE_DATE | 방문일은 오늘 이전 날짜를 선택해 주세요. | 클라이언트 측 날짜 선택기 제한 |
| 좌표 범위 초과 | INVALID_COORDINATE | 유효하지 않은 위치 정보입니다. | 지도 API 통해 입력 받으므로 사실상 발생 최소화, 서버 측 재검증 |
| 사진 개수 초과 | PHOTO_LIMIT_EXCEEDED | 사진은 최대 5장까지 등록 가능합니다. | 클라이언트 측 업로드 버튼 비활성화 |
| 존재하지 않는 장소 | PLACE_NOT_FOUND | 장소를 찾을 수 없습니다. | 404 처리 후 목록 화면으로 이동 |
| 타인 장소 수정 시도 | FORBIDDEN | 본인의 장소만 수정할 수 있습니다. | 403 처리 |

#### 카테고리 (CATEGORY)

| 상황 | 오류 코드 | 표시 메시지 | 처리 방법 |

## 7. 용어 정의

| 용어 | 정의 |
|------|------|
| 장소 (Place) | 사용자가 방문하여 기록한 위치 정보 단위 |
| 마커 (Marker) | 지도 위에 장소 위치를 나타내는 핀/아이콘 |
| 클러스터링 (Clustering) | 인접한 다수의 마커를 하나의 그룹으로 묶어 표시하는 지도 기법 |
| 카테고리 (Category) | 장소를 분류하기 위한 레이블 (예: 맛집, 카페 등) |
| 태그 (Tag) | 장소에 자유롭게 부여하는 키워드 레이블 |
| 소프트 딜리트 (Soft Delete) | 데이터를 물리적으로 삭제하지 않고 삭제 플래그를 설정하는 방식 |
| Access Token | 단기 인증 토큰 (유효 기간 30분) |
| Refresh Token | 장기 인증 토큰 (유효 기간 7일), Access Token 재발급에 사용 |
