## 1. 전체 메뉴 구조도

```
발자국 (Foot-Print)
│
├── [비회원 접근 허용]
│   ├── 로그인                  (SCR-AUTH-01)
│   └── 회원가입                (SCR-AUTH-02)
│
└── [로그인 필요]
    ├── 지도                    (SCR-MAP-01)         ← 기본 랜딩 페이지
    ├── 발자국 목록
    │   ├── 목록 화면           (SCR-PLACE-01)
    │   ├── 장소 상세           (SCR-PLACE-02)
    │   ├── 장소 등록           (SCR-PLACE-03)
    │   └── 장소 수정           (SCR-PLACE-04)
    ├── 통계                    (SCR-STAT-01)
    ├── 카테고리 관리           (SCR-CATEGORY-01)
    └── 마이페이지              (SCR-MYPAGE-01)
```

## 3. 공통 레이아웃

### 3.1 GNB (Global Navigation Bar) - 로그인 후

```
┌────────────────────────────────────────────────────────────┐
│  🗺 발자국     [지도] [목록] [통계]          [프로필 아이콘 ▼] │
└────────────────────────────────────────────────────────────┘
```

| 영역 | 구성 요소 | 동작 |
|------|----------|------|
| 로고 | "발자국" 텍스트 + 아이콘 | 클릭 시 `/map` 이동 |
| 메인 네비게이션 | 지도, 목록, 통계 | 각 화면으로 이동 |
| 프로필 드롭다운 | 마이페이지, 카테고리 관리, 로그아웃 | 드롭다운 메뉴 |

**반응형 분기**

| 구간 | 처리 방식 |
|------|----------|
| PC (1280px+) | GNB 전체 텍스트 메뉴 표시 |
| 태블릿 (768~1279px) | 아이콘 + 텍스트 축약 |
| 모바일 (767px-) | 햄버거 메뉴로 전환, 하단 탭 내비게이션 추가 |

### 3.2 공통 알림 (Toast / Modal)

| 유형 | 사용 상황 | 표시 위치 |
|------|----------|----------|
| 성공 토스트 (초록) | 저장 완료, 삭제 완료 등 | 화면 우상단, 3초 후 자동 사라짐 |
| 오류 토스트 (빨강) | API 오류, 네트워크 오류 | 화면 우상단, 5초 후 자동 사라짐 |
| 정보 토스트 (파랑) | 안내 메시지 | 화면 우상단, 3초 후 자동 사라짐 |
| 확인 모달 | 삭제, 탈퇴 등 파괴적 작업 전 | 화면 중앙 오버레이 |

### SCR-AUTH-01: 로그인

**기본 정보**

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-AUTH-01 |
| 화면명 | 로그인 |
| URL | `/login` |
| 접근 권한 | 비회원 (로그인 상태에서 접근 시 `/map`으로 리다이렉트) |
| 연관 기능 요구사항 | FR-AUTH-02 |

**화면 레이아웃**

```
┌──────────────────────────────────┐
│                                  │
│         🗺 발자국                 │
│       나의 발자국을 기록하세요      │
│                                  │
│  ┌────────────────────────────┐  │
│  │ 이메일                      │  │
│  └────────────────────────────┘  │
│  ┌────────────────────────────┐  │
│  │ 비밀번호                    │  │
│  └────────────────────────────┘  │
│                                  │
│  [        로그인        ]         │
│                                  │
│  계정이 없으신가요?  [회원가입]     │
│                                  │
└──────────────────────────────────┘
```

**구성 요소**

| 컴포넌트 | 타입 | 필수 | 유효성 | 설명 |
|----------|------|------|--------|------|
| 이메일 입력 | text input | Y | 이메일 형식 | placeholder: "이메일을 입력해 주세요" |
| 비밀번호 입력 | password input | Y | 공백 불가 | placeholder: "비밀번호를 입력해 주세요", 표시/숨김 토글 버튼 |
| 로그인 버튼 | primary button | - | - | 클릭 시 API 호출, 로딩 스피너 표시 |
| 회원가입 링크 | text link | - | - | `/register` 이동 |

**동작 흐름**

```
[이메일/비밀번호 입력] → [로그인 버튼 클릭]
        ↓
[클라이언트 유효성 검사]
        ↓ 실패
[필드 하단 오류 메시지 인라인 표시]
        ↓ 성공
[POST /api/v1/auth/login 호출]
        ↓ 실패 (401)
[오류 토스트: "이메일 또는 비밀번호가 올바르지 않습니다."]
        ↓ 성공 (200)
[토큰 저장 → /map 이동]
```

**오류 메시지**

| 상황 | 표시 위치 | 메시지 |
|------|----------|--------|
| 이메일 미입력 | 이메일 필드 하단 | 이메일을 입력해 주세요. |
| 이메일 형식 오류 | 이메일 필드 하단 | 올바른 이메일 형식을 입력해 주세요. |
| 비밀번호 미입력 | 비밀번호 필드 하단 | 비밀번호를 입력해 주세요. |
| 인증 실패 | 토스트 | 이메일 또는 비밀번호가 올바르지 않습니다. |

**기본 정보**

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-AUTH-02 |
| 화면명 | 회원가입 |
| URL | `/register` |
| 접근 권한 | 비회원 |
| 연관 기능 요구사항 | FR-AUTH-01 |

**화면 레이아웃**

```
┌──────────────────────────────────┐
│                                  │
│         🗺 발자국 회원가입         │
│                                  │
│  ┌────────────────────────────┐  │
│  │ 이메일                      │  │
│  └────────────────────────────┘  │
│  ┌────────────────────────────┐  │
│  │ 닉네임                      │  │
│  └────────────────────────────┘  │
│  ┌────────────────────────────┐  │
│  │ 비밀번호                    │  │
│  └────────────────────────────┘  │
│  ┌────────────────────────────┐  │
│  │ 비밀번호 확인                │  │
│  └────────────────────────────┘  │
│                                  │
│  [        회원가입        ]       │
│                                  │
│  이미 계정이 있으신가요?  [로그인] │
│                                  │
└──────────────────────────────────┘
```

**구성 요소**

| 컴포넌트 | 타입 | 필수 | 유효성 규칙 |
|----------|------|------|------------|
| 이메일 입력 | text input | Y | 이메일 형식, 최대 255자 |
| 닉네임 입력 | text input | Y | 2~20자 |
| 비밀번호 입력 | password input | Y | 8~20자, 영문+숫자+특수문자 포함 |
| 비밀번호 확인 입력 | password input | Y | 비밀번호와 일치 |
| 회원가입 버튼 | primary button | - | 모든 필드 유효 시 활성화 |

**동작 흐름**

```
[폼 입력] → [회원가입 버튼 클릭]
        ↓
[클라이언트 유효성 검사]
        ↓ 실패 → [인라인 오류 메시지]
        ↓ 성공
[POST /api/v1/auth/register 호출]
        ↓ 실패 (409 이메일 중복)
[이메일 필드 하단: "이미 사용 중인 이메일입니다."]
        ↓ 성공 (201)
[성공 토스트: "회원가입이 완료되었습니다."] → [/login 이동]
```

**기본 정보**

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-MAP-01 |
| 화면명 | 지도 메인 |
| URL | `/map` |
| 접근 권한 | 로그인 필요 |
| 연관 기능 요구사항 | FR-MAP-01, FR-MAP-02, FR-MAP-03 |

**화면 레이아웃**

```
┌─────────────────────────────────────────────────────────────┐
│ GNB                                                         │
├─────────────────────────────────────────────────────────────┤
│  [카테고리 필터 ▼]  [날짜 범위 ▼]         [+ 장소 등록]       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                                                             │
│                        지도 영역                             │
│                   (Kakao / Google Maps)                     │
│              ★ ★            ★                               │
│                    ★★                                       │
│                                                             │
│                                         [현재 위치 버튼]     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**구성 요소**

| 컴포넌트 | 설명 |
|----------|------|
| 카테고리 필터 드롭다운 | 카테고리 다중 선택, 선택 시 해당 카테고리 마커만 표시 |
| 날짜 범위 필터 | 시작일~종료일 선택, 해당 기간 방문 장소만 표시 |
| 장소 등록 버튼 | 클릭 시 SCR-PLACE-03으로 이동 (좌표 미선택 상태) |
| 지도 | 전체 화면 지도, 마커/클러스터링 표시 |
| 현재 위치 버튼 | 브라우저 Geolocation API로 현재 위치로 지도 이동 |

**마커 팝업 상세**

지도에서 마커 클릭 시 팝업 표시:

```
┌──────────────────────────────┐
│ [썸네일 사진]  장소명           │
│               카테고리 뱃지    │
│               방문일: 2026.04.15 │
│                              │
│  [상세 보기 →]               │
└──────────────────────────────┘
```

**동작 흐름**

```
[화면 진입]
    ↓
[GET /api/v1/places?view=map 호출]
    ↓
[지도 위에 마커 렌더링]
    ↓
[마커 클릭] → [팝업 표시]
    ↓
[팝업 > 상세 보기] → [SCR-PLACE-02]
    ↓
[지도 클릭 (빈 공간)] → [해당 좌표로 SCR-PLACE-03 진입]
    ↓
[필터 변경] → [필터 파라미터로 API 재호출 → 마커 갱신]
```

```
│              │  ┌────────────────────────────────────────┐  │
│ 카테고리      │  │ [썸네일] 장소명          방문일          │  │

│              │                                              │

```

**구성 요소**

| 컴포넌트 | 설명 |
|----------|------|
| 검색 입력창 | 실시간 검색 (debounce 300ms 적용), 장소명/주소/메모/태그 검색 |
| 장소 등록 버튼 | SCR-PLACE-03으로 이동 |
| 필터 패널 | 카테고리 다중 선택, 날짜 범위, 평점 필터 |
| 정렬 드롭다운 | 방문일 최신순/오래된순, 등록일순, 평점 높은순 |
| 장소 카드 | 썸네일, 장소명, 카테고리, 방문일, 주소, 평점, 메모 미리보기 |
| 결과 건수 표시 | "총 N개" 표시 |
| 빈 목록 안내 | 검색/필터 결과 없을 시 "등록된 장소가 없습니다." 안내 |

**동작 흐름**

```
[화면 진입]
    ↓
[GET /api/v1/places 호출 (기본: 방문일 최신순, 20건)]
    ↓
[장소 카드 목록 렌더링]
    ↓
[카드 클릭] → [SCR-PLACE-02]
[검색어 입력] → [debounce 후 API 재호출]
[필터 변경 > 적용] → [필터 파라미터 포함 API 재호출]
[정렬 변경] → [정렬 파라미터 포함 API 재호출]
[스크롤 하단 도달] → [다음 페이지 API 호출 (무한 스크롤)]
```

**반응형 처리**

| 구간 | 카드 레이아웃 | 필터 패널 |
|------|-------------|----------|
| PC | 사이드 필터 패널 + 카드 2열 | 항상 노출 |
| 태블릿 | 카드 2열 | 접힘/펼침 토글 버튼 |
| 모바일 | 카드 1열 | 하단 슬라이드업 패널 |
