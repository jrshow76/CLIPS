# Memory Card Game 기능 명세서 (SRS)

- 프로젝트명: Memory Card Game (짝맞추기 카드 게임)
- 게임 유형: 1인용, 같은 그림 카드 짝맞추기 (Memory / Concentration)
- 기술 스택: Vanilla HTML/CSS/JS + Phaser 3 (CDN 로드, 빌드 도구 없음)
- 문서 버전: v1.0 / 2026-05-11

---

## 1. 게임 화면 흐름

```
[시작 화면] → [게임 화면] → [결과 화면]
     ↑              ↓              ↓
     └──── 메뉴 ────┴──── 재시작 ──┘
```

1. **시작 화면(Title)**: 게임 제목, 난이도 선택(Easy/Normal/Hard), 최고기록 표시, [게임 시작] 버튼
2. **게임 화면(Play)**: 카드 보드, 타이머, 시도 횟수, 점수, [재시작] / [메뉴] 버튼
3. **결과 화면(Result)**: 클리어 시간, 시도 횟수, 최종 점수, 신기록 여부, [다시 하기] / [난이도 변경] 버튼

---

## 2. 게임 룰

| 항목 | 규칙 |
|---|---|
| 카드 배치 | 난이도별 카드 수만큼 이모지 풀에서 무작위 추출 후 쌍을 만들어 셔플 |
| 카드 뒤집기 | 클릭 시 카드 앞면(이모지) 공개. 한 번에 최대 2장만 공개 가능 |
| 매칭 판정 | 2장의 이모지가 동일하면 매칭 성공(앞면 고정), 다르면 매칭 실패 |
| 잠금 시간 | 매칭 실패 시 800ms 동안 입력 잠금 후 두 카드 다시 뒤집기 |
| 시도 횟수 | 2장째 카드를 뒤집은 시점에 +1 (성공/실패 모두 카운트) |
| 클리어 조건 | 모든 카드가 매칭 성공 상태가 되면 결과 화면 전환 |
| 타이머 | 첫 카드 뒤집는 순간 시작, 클리어 시점에 정지 (mm:ss) |

### 난이도

| 난이도 | 보드 | 카드 수 | 쌍 수 |
|---|---|---|---|
| Easy | 4 x 3 | 12 | 6 |
| Normal | 4 x 4 | 16 | 8 |
| Hard | 6 x 4 | 24 | 12 |

## 3. 점수 계산 공식

매칭 성공 시점마다 가점, 실패 시 감점 없음. 최종 점수 = 매칭 점수 합 - 시도 페널티 + 시간 보너스.

- **매칭 기본 점수**: `100점`
- **연속 매칭 보너스(콤보)**: 매칭 성공이 연달아 발생할 때마다 `+ 50 x (콤보-1)` (실패 시 콤보 0으로 초기화)
- **시도 페널티**: `-5점 x max(0, 시도횟수 - 쌍수)` (최소 시도 = 쌍수 기준 초과분만 감점)
- **시간 보너스**: `max(0, (기준시간 - 경과초) x 난이도계수)`
  - 기준시간: Easy 60초 / Normal 100초 / Hard 180초
  - 난이도계수: Easy 2 / Normal 3 / Hard 5
- **신기록 판정**: 동일 난이도 기준 최고 점수 갱신 시 표시

```
최종 점수 = (100 x 쌍수) + 콤보보너스 합 - 5 x max(0, 시도-쌍수) + 시간보너스
```

## 4. 화면별 UI 요소

### 4.1 시작 화면
- 타이틀 로고, 난이도 라디오 버튼 3종, 난이도별 최고기록 카드(점수/시간/시도), [게임 시작], 사운드 ON/OFF 토글

### 4.2 게임 화면
- 상단 HUD: 타이머(mm:ss), 시도 횟수, 현재 점수, 콤보 인디케이터
- 중앙: 카드 그리드 (난이도별 NxM)
- 하단: [재시작], [메뉴로]
- 카드 상태: 뒷면(기본) / 앞면(공개) / 매칭완료(비활성·하이라이트)

### 4.3 결과 화면
- 결과 메시지 ("Clear!"), 소요 시간, 시도 횟수, 최종 점수, 신기록 배지(조건부), [다시 하기], [난이도 변경]

## 5. 데이터 모델 (localStorage)

| Key | 값 형식 | 설명 |
|---|---|---|
| `mcg.bestScore.easy` | number | Easy 최고 점수 |
| `mcg.bestScore.normal` | number | Normal 최고 점수 |
| `mcg.bestScore.hard` | number | Hard 최고 점수 |
| `mcg.bestTime.easy` | number(sec) | Easy 최단 시간 |
| `mcg.bestTime.normal` | number(sec) | Normal 최단 시간 |
| `mcg.bestTime.hard` | number(sec) | Hard 최단 시간 |
| `mcg.settings` | JSON | `{ soundOn: boolean, lastDifficulty: "easy"\|"normal"\|"hard" }` |

- 저장 시점: 결과 화면 진입 시. 기존 값보다 좋은 경우에만 갱신.
- 손상 데이터 대응: JSON 파싱 실패 시 기본값으로 초기화.

## 6. 예외 처리 정책

| 케이스 | 처리 방식 |
|---|---|
| 빠른 연속 클릭 | 입력 잠금 플래그(`isLocked`)로 2장 초과 뒤집기 차단 |
| 이미 공개된 카드 재클릭 | 무시 (이벤트 소비만) |
| 매칭 완료 카드 클릭 | 무시 |
| 매칭 실패 후 자동 복귀 | 800ms 딜레이 후 두 카드 동시 뒤집기, 딜레이 중 입력 차단 |
| 게임 중 [재시작] | 확인 다이얼로그 → 동일 난이도로 새 보드 생성 |
| localStorage 사용 불가 | 메모리 임시 저장으로 폴백, 경고 토스트 표시 |
| 사운드 로드 실패 | 무음 처리하고 토글은 유지 |
| 창 비활성화(blur) | 타이머는 계속 진행 (실시간 기준) — 일시정지 미지원 |

## 9. 비기능 요구사항

- 반응형: 최소 360px 폭 지원 (모바일 세로), 카드 그리드 자동 스케일
- 접근성: 카드에 `aria-label` 부여(예: "카드 1, 뒤집힘"), 키보드 포커스 가능
- 성능: 카드 뒤집기 애니메이션 200ms 이내, 60fps 유지
- 브라우저 지원: 최신 Chrome / Safari / Firefox / Edge
