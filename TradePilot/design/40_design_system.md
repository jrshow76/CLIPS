# TradePilot 디자인 시스템

> 문서 ID: 40_DESIGN_SYSTEM
> 버전: v1.0
> 작성자: Designer
> 최종 수정일: 2026-05-12

본 문서는 TradePilot의 디자인 시스템(컬러/타이포/간격/반응형/컴포넌트 원칙)을 정의한다. 모든 토큰의 단일 출처는 `tokens/tokens.css`이며, 본 문서는 해당 토큰을 시각적으로 정리한 참고서이다.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|---|---|
| 데이터 우선 (Data-first) | 시각적 장식보다 숫자/차트의 가독성을 우선한다 |
| 모드 가시성 (Mode visibility) | LIVE/SIM 상태는 항상 화면에 명시적으로 표시한다 |
| 일관된 등락색 | 상승=빨강, 하락=파랑 컨벤션을 모든 화면에서 일관 적용 |
| 다크 우선 (Dark-first) | 장시간 사용에 유리한 다크 모드를 기본값으로 |
| 즉시 식별 (At-a-glance) | KPI/배지/시그널은 0.5초 안에 의미를 파악할 수 있어야 함 |
| 안전한 모드 전환 | LIVE 전환·비상정지는 별도 시각 신호(빨강 배경/배너) |

## 2. 색상 토큰

### 2.1 브랜드 / 액션
| 토큰 | 값(다크) | 용도 |

|---|---|---|

### 2.4 사용 가이드
- 등락 표시는 반드시 색 + 부호(`+`, `-`) + 화살표(▲▼)를 함께 노출하여 색맹 사용자 접근성 보장.
- 모드 배지: SIM = `badge--sim` (파랑), LIVE = `badge--live` (빨강 + 점멸 없음). 비상시는 상단 `banner--live` 추가.

## 3. 타이포그래피

| 토큰 | 크기 | 용도 |
|---|---|---|
| `--fs-12` | 12px | 캡션, 표 헤더 |
| `--fs-13` | 13px | 표 본문, 보조 텍스트 |
| `--fs-14` | 14px | **본문 기본** |
| `--fs-16` | 16px | 카드 제목 |
| `--fs-20` | 20px | 페이지 보조 제목 |
| `--fs-24` | 24px | 페이지 제목, KPI 값 |
| `--fs-32` | 32px | 강조 KPI(대시보드 메인) |

- 본문 폰트: `Pretendard` → 시스템 sans 폴백.
- 숫자: `font-variant-numeric: tabular-nums`를 적용한 `.text-num` 또는 `.text-mono`로 정렬 일관성.
- 한글 줄간격: `--lh-normal 1.45`. 표 안에서는 `--lh-tight 1.2`.

## 4. 간격 / 그리드

### 4.1 간격 스케일 (4px 기준)
`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64` → 토큰 `--space-1` ~ `--space-16`.

### 4.2 컨테이너 / 레이아웃
| 항목 | 값 |
|---|---|
| 페이지 최대 폭 | `1440px` (`--container-max`) |
| 사이드바 폭 | `240px` (collapsed `64px`) |
| 헤더 높이 | `56px` |
| 카드 기본 패딩 | `20px` |
| 카드 헤더 패딩 | `16px 20px` |
| 테이블 셀 패딩 | `12px 16px` |

### 4.3 그리드 유틸리티
`.grid-cols-2 / 3 / 4 / 12` 제공. 1024px 이하에서 4→2, 3→2 자동 축소, 768px 이하에서 1열 스택.

---

## 5. 반응형 Breakpoint

| 디바이스 | 폭 | 그리드 정책 |
|---|---|---|
| Desktop | ≥ 1280px | 사이드바 풀폭, 3컬럼 대시보드 |
| Laptop  | ≥ 1024px | 사이드바 축소(아이콘), 3컬럼 유지 |
| Tablet  | ≥ 768px  | 사이드바 숨김, 2컬럼 |
| Mobile  | < 768px  | 1컬럼, 헤더 햄버거 메뉴, 테이블 → 카드 변환 |

미디어 쿼리:
```css
@media (max-width: 1280px) { /* laptop down */ }
@media (max-width: 1024px) { /* tablet down */ }
@media (max-width:  768px) { /* mobile */ }
```

## 7. 컴포넌트 원칙

- BEM (`block__element--modifier`) + CSS 변수.
- 컴포넌트는 **자체 패딩/마진을 최소화**하고, 컨테이너가 간격을 책임진다.
- 상태(hover/active/disabled/loading/empty/error) 5종은 모든 컴포넌트에서 정의.
- React 변환 시 모든 BEM 클래스는 컴포넌트 파일 1:1 매핑이 가능하도록 작성.

### 7.1 표준 상태
| 상태 | 처리 |
|---|---|
| 로딩 | `.skeleton` 블록 또는 spinner. 5초 초과 시 진행률 표시. |
| 빈 상태 | `.empty-state` (아이콘 + 메시지 + CTA) |
| 부분 장애 | `.error-card` (재시도 버튼) |
| 시세 지연 | `.badge badge--warning` + "지연" 텍스트 |
| 장 종료 | 상단 `.banner banner--warning` + 매매 버튼 비활성 |

## 8. 접근성

| 항목 | 기준 |
|---|---|
| 본문 대비 | 4.5:1 이상 |
| 키보드 포커스 | `:focus-visible` 시 `--shadow-focus` 링 |
| 색상 단독 의존 금지 | 등락 색 + 부호/화살표 함께 |
| 모달 | `role="dialog"`, `aria-modal="true"`, `Esc`로 닫기 |
| 단축키 | `Ctrl+K` 검색, `Ctrl+Shift+S` 비상정지, `Ctrl+M` 모드 토글, `Esc` 모달 |

## 9. 아이콘 / 일러스트

- 아이콘 셋: lucide (또는 heroicons). 본 마크업에서는 SVG가 들어갈 자리에 의미 텍스트(예: `→`, `★`)나 한글 약어를 사용.
- 일러스트 자리는 회색 placeholder 원형 사용(`.empty-state__icon`).

## 10. 차트 / 데이터 시각화 가이드

| 요소 | 권장 |
|---|---|
| 캔들차트 | 라이브러리 lightweight-charts 또는 echarts. 색: 상승=빨강, 하락=파랑 |
| 라인차트 | 1차 시계열은 `--color-brand-500`, 비교 시리즈는 보조 색상 사용 |
| 히트맵 | 빨강/파랑 단계 8단계 (현재 마크업은 `.heatmap__cell--up/down-1~3` 정의) |
| 게이지 / 진행률 | `.progress` 사용 |
| 차트 그리드 | `--chart-grid-1/2` 토큰 사용, 다크에서는 흰색 4%/8% |
