본 문서는 `12_screen_flow.md`의 IA 트리에 따른 36개 화면을 디자이너 관점에서 정리한다. 화면별로 와이어 설명, 표시 데이터, 상태 변형(empty/loading/error), 우선 인터랙션을 명세한다.

화면은 **메인 8 + 서브 28 = 36개**이며 분류는 다음과 같다.

| 분류 | 메인 화면 | 서브 화면 | 총 |
|---|:---:|:---:|:---:|
| 인증 | 1 | 2 | 3 |
| 대시보드 | 1 | 0 | 1 |
| 추천주 | 1 | 1 | 2 |
| 차트분석 | 1 | 1 | 2 |
| 업종분석 | 1 | 2 | 3 |
| 시그널 | 1 | 2 | 3 |
| 자동매매 | 1 | 4 | 5 |
| 리포트 | 1 | 3 | 4 |
| 백테스트 | 0 | 3 | 3 |
| 설정 | 0 | 4 | 4 |
| 공통(오류/상태) | 0 | 6 | 6 |
| 합계 | **8** | **28** | **36** |

## 1. 인증 (3개)

### S-AUTH-01 로그인 `pages/login.html`
- 와이어: 좌(브랜드 일러스트/슬로건), 우(로그인 폼). 모바일에서는 1컬럼 스택.
- 데이터: email, password, 자동 로그인 체크, 비밀번호 재설정 링크.
- 상태:
  - loading: 버튼 내부 스피너.
  - error: 인풋 하단 빨강 메시지 + `error-card` (E0001, E0052).

### S-AUTH-02 회원가입
- 와이어: 단일 컬럼 폼. email/password/닉네임/약관 체크.
- 상태: 이메일 중복 E0051 인라인, 비밀번호 정책 E0055는 입력 시 실시간 가이드.

### S-AUTH-03 비밀번호 재설정
- 와이어: 단일 컬럼. 이메일 입력 → 메일 발송 안내 → 토큰 입력 → 새 비번 설정의 3단계 스텝퍼.

## 2. 대시보드 (1개) `pages/dashboard.html`

### S-DASH-01 메인 대시보드
- 와이어 (Desktop 3컬럼):
  - 좌: 보유 종목 카드 리스트 + 일일 손익 위젯
  - 중: KOSPI/KOSDAQ 지수, 미체결 시그널 카드, 모드 배지
  - 우: 추천주 TOP5, 최근 알림
- 모바일: 좌 → 중 → 우 순으로 스택.
- 상태:
  - 보유 종목 0건: `empty-state` "시뮬레이션 매수로 시작하기" CTA
  - 시세 지연: 가격 옆 `badge--warning` "지연"
  - LIVE 모드: 상단 `banner--live` 노출

## 3. 추천주 (2개)

### S-RECO-01 추천 리스트 `pages/recommendations.html`
- 와이어: 상단 필터바(섹터/시총/RSI/MACD/정렬), 본문 테이블(종목/점수/현재가/등락률/추천사유/액션).
- 모바일: 테이블 → 카드 리스트로 변환.
- 상태: 0건 시 빈 상태 + "조건 완화" CTA, 점수 정렬 toggle.

### S-RECO-02 종목 상세 (모달/페이지)
- 와이어: 좌(메타정보, 차트 미니), 우(지표 카드 + ML 예측 + 뉴스 키워드 Top10).
- 상태: 뉴스 소스 장애 E0004 시 키워드 영역만 `error-card`, 다른 영역은 정상.

## 4. 차트분석 (2개)

### S-CHART-01 캔들/지표 `pages/chart-analysis.html`
- 와이어:
  - 상단: 종목 검색, 주기 토글(D/W/M/1m/5m/15m/30m), 지표 추가
  - 메인: 캔들 차트(`chart-placeholder--lg`), 하단 보조: 거래량/RSI/MACD 패널
  - 우측 사이드(Desktop ≥1280): 지표 파라미터 패널
- 모바일: 사이드 패널 → 하단 드로어로.
- 상태: 데이터 누락 구간 점선(설명 자리표시), PNG 다운로드 버튼.

### S-CHART-02 2종목 비교
- 와이어: 종목 A/B 선택 → 정규화(시작값=100) 차트 + 상관계수 카드.

## 6. 시그널 (3개)

### S-SIGNAL-01 활성 시그널 `pages/signals.html`

## 7. 자동매매 (5개)

### S-AUTO-01 전략 목록 `pages/auto-trading.html`
- 와이어: 좌(전략 카드/리스트), 우(전략 상세). 활성 토글, 손익 미니 차트.
- 상단 메뉴: [전략 목록] [등록/수정] [모드 전환] [한도/리스크] [비상정지]

### S-AUTO-02 전략 등록/수정
- 와이어: 폼 그리드 2컬럼. 진입조건/청산조건/유니버스/한도/스케줄.

### S-AUTO-03 모드 전환 (`MD-001` 모달 흐름)
- 단계: 약관(`MD-002`) → OTP(`MD-003`) → 한도 설정 → 크레온 연결 → 최종 확인.
- 시각: 최종 확인 모달은 `modal--danger`, 빨강 상단 라인 + 체크박스 강제.

### S-AUTO-04 한도/리스크
- 와이어: 일일 매수금액/건수, 종목당 한도, 손절/익절% 입력. 슬라이더 + 인풋 조합.

### S-AUTO-05 비상정지 (`MD-004`)
- 와이어: 큰 빨강 버튼 + 2단계 확인 모달. 결과 카드(취소 N건, 실패 M건).

## 12. 상태별 변형 매트릭스

모든 데이터 영역은 3종 상태를 가진다.

| 상태 | 처리 | 적용 컴포넌트 |
|---|---|---|
| Loading | `.skeleton`을 표/카드 영역에 배치, KPI는 `--` 표기 | `.skeleton--line`, `.skeleton--text` |
| Empty   | 일러스트(원형) + 메시지 + CTA | `.empty-state` |
| Error   | 빨강 보더 카드 + 재시도 버튼 | `.error-card` |

예시 적용 화면:
- 대시보드 보유 종목: 0건 → empty, 시세 장애 → 우측 추천주 영역만 error, 다른 카드는 정상.
- 차트분석: 누락 구간은 차트 내 점선(설명) + 우측 상태 텍스트.
- 백테스트 결과: 만료(E0031) → empty + "재실행" CTA.
