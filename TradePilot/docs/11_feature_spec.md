# TradePilot 기능 정의서 (Feature Specification)

> 문서 ID: 11_FEATURE_SPEC
> 버전: v1.0
> 작성자: Planner
> 최종 수정일: 2026-05-12

본 문서는 화면/기능 단위로 **입력 → 처리 → 출력 → 예외** 흐름을 정의한다. 각 기능 ID는 SRS의 FR-ID와 매핑된다.

---

## 0. 공통 사항

### 0.1 공통 헤더 / 환경
| 항목 | 내용 |
|---|---|
| 인증 | JWT `Authorization: Bearer {token}` |
| 모드 헤더 | `X-Trade-Mode: SIM | LIVE` (자동매매/주문 관련 호출 시 필수) |
| 사용자 컨텍스트 | `X-User-Id` (게이트웨이에서 자동 주입) |
| 시간대 | KST(Asia/Seoul), 모든 timestamp는 ISO-8601 |

### 0.2 공통 응답 포맷
```
성공: { "success": true,  "data": {...} }
실패: { "success": false, "error": { "code": "E1001", "message": "..." } }
페이지: { "success": true, "data": { "items": [...], "page": 1, "size": 20, "total": 145 } }
```

### 0.3 공통 예외(전역)
| 코드 | 설명 |
|---|---|
| E0001 | 인증 실패 / 토큰 만료 |
| E0002 | 권한 부족 |
| E0003 | 파라미터 검증 실패 |
| E0004 | 외부 시스템 일시 장애 |
| E0005 | 서버 내부 오류 |
| E0006 | 매매 모드 불일치(`X-Trade-Mode`와 사용자 설정 불일치) |
| E0007 | 장 운영시간 외 요청 거부 |

## 1. 대시보드 (FR-DASH)

### 1.1 화면 구성
- 좌측: 보유 종목 카드 리스트, 일일 손익 위젯
- 중앙: 코스피/코스닥 지수, 진행중 시그널, 매매 모드 배지(SIM/LIVE)
- 우측: 추천주 TOP5, 최근 알림

### 1.2 기능 표
| 기능 ID | 기능명 | 입력 | 처리 | 출력 | 예외 |
|---|---|---|---|---|---|
| F-DASH-01 | 보유종목 조회 | 사용자 ID(헤더) | `portfolios` 조회 + 현재가 조인 | 종목/수량/평단/평가손익/손익률 | E0001, E0004(시세 미수신 시 -- 표시) |
| F-DASH-02 | 일일 손익 요약 | 사용자 ID, 기준일(KST today) | 당일 체결+평가손익 합산 | 당일 손익, 누적 손익 | E0001 |
| F-DASH-03 | 시장 요약 | 없음 | KOSPI/KOSDAQ 지수 polling(5s) | 지수, 등락률, 거래대금 | E0004 → 직전 캐시 노출 |
| F-DASH-04 | 추천주 TOP5 | 사용자의 활성 전략 ID 목록 | `recommendations`에서 score DESC TOP5 | 종목/전략명/점수/사유 | 결과 0건 시 빈 상태 UI |
| F-DASH-05 | 진행중 시그널 | 사용자 ID | `signals` where status=ACTIVE | 시그널 카운트, 미체결 주문 | E0001 |
| F-DASH-06 | 모드 배지 | `users.trade_mode` | 화면 우상단 배지 표시 | SIM(파랑) / LIVE(빨강) | - |

## 2. 추천주 (FR-RECO)

### 2.1 기능 표
| 기능 ID | 기능명 | 입력 | 처리 | 출력 | 예외 |
|---|---|---|---|---|---|
| F-RECO-01 | 추천 리스트 조회 | 전략 ID(선택), 섹터, 시총구간, 지표 필터, 정렬, page, size | 추천 엔진 결과를 페이지네이션 | 종목 리스트(점수, 사유, 현재가, 등락률) | E0003 |
| F-RECO-02 | 종목 상세 | 종목코드 | 기본정보 + 지표 + ML 예측 + 뉴스 키워드 집계 | 종목 메타, 지표값, 예측치, 뉴스 키워드 Top10 | E0003, E0004(뉴스 소스 장애 시 키워드 생략) |
| F-RECO-03 | 즐겨찾기 등록/해제 | 종목코드 | `user_favorites` upsert/delete | 등록 결과 | E0001 |
| F-RECO-04 | 추천 사유 표시 | 추천 레코드 ID | rule_id 매칭 → 템플릿 텍스트 생성 | "RSI<30 + 거래량 200% 증가" 형식 텍스트 | - |

|---|---|---|

| `market_cap_min` / `market_cap_max` | int (억 원) | 범위 |
| `rsi_max` / `rsi_min` | float | 0~100 |
| `macd_signal` | enum | `GOLDEN`, `DEAD`, `NONE` |
| `volume_change_pct` | float | 전일대비 거래량 증가율 |
| `sort` | enum | `SCORE_DESC`, `CHANGE_DESC`, `VOLUME_DESC` |

## 9. 백테스트 (FR-BACKTEST)

| 기능 ID | 기능명 | 입력 | 처리 | 출력 | 예외 |
|---|---|---|---|---|---|
| F-BT-01 | 백테스트 실행 | 전략 ID, 종목 유니버스, 기간, 초기자본, 슬리피지, 수수료 | 작업 큐 enqueue → 워커 비동기 실행 | job_id, 진행률 | E0003 |
| F-BT-02 | 진행률 조회 | job_id | 워커 상태 조회 | percent, ETA | E0003 |
| F-BT-03 | 결과 조회 | job_id | 결과 집계 | 누적·연환산 수익률, MDD, 승률, 샤프, 거래 타임라인 | E0003, E0031(결과 만료) |
| F-BT-04 | 결과 비교 | job_id[] (최대 5) | 결과 정규화 후 동일 차트에 오버레이 | 비교 차트 데이터 | E0003 |
| F-BT-05 | 결과 저장 | job_id, label | `backtest_results`에 라벨링 저장 | 결과 | - |

### 9.1 백테스트 검증 규칙
- 기간 최대 5년, 종목 유니버스 최대 200개.
- 초기자본 ≥ 1,000,000원, 슬리피지 ≥ 0, 수수료 0~1%.
