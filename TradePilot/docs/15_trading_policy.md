# TradePilot 매매 정책 정의서

> 문서 ID: 15_TRADING_POLICY
> 버전: v1.0
> 작성자: Planner
> 최종 수정일: 2026-05-12

본 문서는 TradePilot의 매매 모드 정책, 시뮬레이션/실거래 분기, 주문 한도, 리스크 컨트롤, 강제 청산 조건을 정의한다. 본 문서는 정책 변경 시 PM 승인 후에만 갱신된다.

---

## 1. 매매 모드 개요

### 1.1 모드 종류
| 모드 | 코드 | 설명 |
|---|---|---|
| 시뮬레이션 | `SIM` | 가상 잔고/체결, 외부 주문 발생 없음 (기본값) |
| 실거래 | `LIVE` | CREON COM을 통해 실제 주문 발생 |

### 1.2 모드 저장 위치 / 적용 범위
- 사용자 단위 저장(`users.trade_mode`).
- 모든 주문 관련 API는 요청 헤더 `X-Trade-Mode`와 사용자 설정이 일치해야 처리된다(불일치 시 E0006).
- 자동매매 엔진은 모드별 라우팅 어댑터(SimRouter / LiveRouter)를 통해 분기한다.

### 1.3 책임 분리 (분기 시점)
| 단계 | SIM | LIVE | 책임 컴포넌트 |
|---|---|---|---|
| 시그널 생성 | 동일 | 동일 | Strategy Engine |
| 한도/리스크 검사 | 동일 | 동일 | Risk Guard |
| 주문 라우팅 | SimRouter | LiveRouter | Order Service |
| 체결 처리 | 가상 체결기(슬리피지 0.1%) | CREON COM 응답 수신 | Execution Handler |
| 잔고 갱신 | 가상 portfolio | 실제 portfolio | Portfolio Service |
| 결과 표시 | UI 파랑 배지 | UI 빨강 배지 | Frontend |

## 2. 시뮬레이션 ↔ 실거래 전환 정책

### 2.1 SIM → LIVE 전환 사전 조건 (모두 충족)
| 번호 | 조건 | 검증 방식 | 미충족 시 코드 |
|---|---|---|---|
| 1 | 본인인증(이메일 + 휴대전화) 완료 | `users.email_verified=true` AND `phone_verified=true` | E0016 |
| 2 | OTP 6자리 인증 성공(3분 유효) | `otp_token` 검증 | E0011 |
| 3 | 약관 / 면책 / 리스크 고지 동의 | `disclaimer_agreed_at` 존재 | E0013 |
| 4 | 일일/종목 한도 1회 이상 저장 | `risk_limits` 레코드 존재 | E0016 |
| 5 | 시뮬레이션 누적 거래 ≥ 30건 | `orders` count(mode=SIM) ≥ 30 | E0016 |
| 6 | CREON COM 연결 테스트 성공 | `CpUtil.CpCybos.IsConnect == 1` | E0012 |
| 7 | 모드 전환 동시성 검증 | 동일 사용자 진행 중 전환 없음 | E0017 |

### 2.2 LIVE → SIM 전환 절차
1. 진행 중 자동매매 일시 정지.
2. 미체결 주문 전체 조회 → 일괄 취소 요청.
3. 취소 결과 집계, 부분 실패 시 사용자에게 안내(E0014).
4. `users.trade_mode = SIM`으로 갱신.
5. `audit_log` 기록(주체, 시각, 미체결 처리 내역).

### 2.3 자동 전환 트리거 (시스템 강제)
| 트리거 | 조치 |
|---|---|
| 일일 손실 한도 초과 | LIVE → SIM 강제 + 자동매매 OFF + 알림 |
| CREON 연결 단절(3회 재연결 실패) | LIVE → SIM 강제 + 운영자 알림 |
| Kill Switch 발동 | LIVE → SIM 강제 + 모든 주문 취소 |
| 시스템 메인터넌스 진입 | 모든 사용자 LIVE → SIM 일시 전환 |

## 3. 주문 한도 정책

### 3.1 사용자 단위 기본 한도 (디폴트)
| 항목 | 기본값 | 최대값 | 비고 |
|---|---|---|---|
| 일일 매수 금액 | 5,000,000원 | 100,000,000원 | LIVE 모드 한정 |
| 일일 매수 건수 | 20건 | 100건 | - |
| 종목당 매수 금액 | 1,000,000원 | 30,000,000원 | 분산 강제 |
| 보유 종목 수 | 10개 | 30개 | - |
| 손절선(stop_loss_pct) | -3.0% | -10.0% | 음수 |
| 익절선(take_profit_pct) | +5.0% | +30.0% | - |
| 일일 손실 한도(daily_loss_limit_pct) | -5.0% | -15.0% | 자산 대비 |
| 단일 주문 최대 수량 | 1,000주 | 10,000주 | - |

### 3.2 한도 검증 시점
1. 시그널 → 주문 생성 직전(Risk Guard).
2. 사용자가 수동 주문 클릭 시 클라이언트 + 서버 양쪽.
3. 한도 변경 시 즉시 반영(이미 발주된 주문은 영향 없음).

### 3.3 한도 초과 처리
| 케이스 | 코드 | 처리 |
|---|---|---|
| 일일 매수 금액 초과 | E0021 | 주문 거부, 일별 카운터는 자정 리셋 |
| 일일 매수 건수 초과 | E0021 | 주문 거부 |
| 종목당 한도 초과 | E0021 | 부분 주문 옵션 제공(설정에 따라) |
| 보유 종목 수 초과 | E0021 | 신규 진입 차단, 기존 포지션 추가 매수만 허용 |

## 4. 리스크 컨트롤 정책

### 4.1 사전 리스크 게이트 (Pre-Trade)
- 손절/익절선이 미설정된 시그널은 주문 거부.
- 변동성(ATR 14) > 임계값 종목은 한도 50% 자동 축소.
- 시가총액 ≤ 500억(소형주)은 단일 주문 한도의 30%까지만 허용.
- 신용/대주 잔고 비율 임계값 초과 종목은 매수 차단.

### 4.2 실시간 리스크 모니터링 (In-Trade)
| 모니터 | 주기 | 조치 |
|---|---|---|
| 보유 종목 손익률 | 1초(WS) | 손절/익절 도달 시 시장가 청산 주문 |
| 일일 누적 손익 | 30초 | 손실 한도 도달 시 자동매매 OFF + 강제 청산 |
| 주문 오류율 | 1분 | 5% 초과 시 신규 주문 일시 중지 |
| 슬리피지 | 매 체결 | 임계값 초과 종목 30분 쿨다운 |
| CREON 헬스체크 | 5초 | 연결 단절 시 LIVE→SIM 강제 |

### 4.3 사후 리스크 대응 (Post-Trade)
- 매일 장 종료 후 리스크 리포트 자동 생성(이메일).
- 종목별/전략별 슬리피지·MDD·승률 트렌드 모니터링.
- 임계값 초과 시 운영자 에스컬레이션.

### 4.4 시장 단위 차단
- 코스피/코스닥 일일 등락률 ±5% 이상: 신규 매수 차단.
- 사이드카/서킷브레이커 발동: 모든 매매 일시 정지.
- 환율/금리 급변동(설정 임계값) 시 옵션으로 매매 중단.

## 5. 강제 청산 정책

### 5.1 강제 청산 트리거
| 우선순위 | 트리거 | 청산 범위 |
|---|---|---|
| 1 | 일일 손실 한도 도달 | 전 보유 종목 시장가 매도 |
| 2 | 종목별 손절선 도달 | 해당 종목만 시장가 매도 |
| 3 | 사용자 Kill Switch | 진행 중 주문 취소(보유는 유지) |
| 4 | 시스템 비상정지(운영) | 전 사용자 자동매매 OFF, 보유 유지 |
| 5 | CREON 단절 지속(3회 재연결 실패) | 진행 중 주문 취소(보유 유지) |

### 5.2 강제 청산 절차
1. 트리거 감지 → Risk Guard 잠금(중복 방지).
2. 청산 대상 종목/수량 결정.
3. 시장가 매도 주문 일괄 전송(동시성 5).
4. 결과 집계 → 미처리 종목 재시도(최대 3회).
5. 사용자 알림 발송(채널: 인앱 + 이메일 + (옵션) SMS).
6. `audit_log` 기록 + 자동매매 모드 SIM 강제.

### 5.3 강제 청산 예외
- 거래 정지 종목: 청산 불가, `failed[]`에 사유 포함.
- 상한가/하한가 락: 시장가 거부 시 다음 호가로 1회 재시도, 실패 시 미처리 보고.
- 단일가(시초/마감 동시호가) 시간대: 큐잉 후 정규 시간 진입 시 처리.

## 6. 자동매매 운영 시간

| 시간대(KST) | 동작 |
|---|---|
| 08:30~09:00 | 사전 점검(시세 워밍업, 모델 로딩, 헬스체크) |
| 09:00~15:20 | 정규 매매(시그널/주문 활성) |
| 15:20~15:30 | 신규 매수 차단, 청산만 허용 |
| 15:30~ | 일일 정산, 리포트 생성 |
| 휴장일 | 매매 비활성, 백테스트/분석만 가능 |

- 정규장 외 동시호가 시간(예: 시간외 단일가)는 옵션으로 활성/비활성.

## 7. 수수료 / 세금 계산 정책

| 항목 | 적용 |
|---|---|
| 위탁수수료(LIVE) | CREON 응답값 우선, 없을 시 0.015% |
| 위탁수수료(SIM) | 0.015% 가정 |
| 거래세(매도) | 코스피 0.18%, 코스닥 0.18% (정책 변경 가능, 설정으로 관리) |
| 슬리피지(SIM) | 시장가 ±0.1% |
| 환산 통화 | KRW |

- 백테스트/리포트는 위 값을 기본으로 사용하되, 백테스트 입력에서 사용자 정의 가능.

## 8. 알림 정책

| 이벤트 | 우선순위 | 채널 |
|---|---|---|
| 강제 청산 | CRITICAL | 인앱 + 이메일 + SMS(옵션) |
| 일일 손실 한도 도달 | HIGH | 인앱 + 이메일 |
| 매수/매도 체결 | NORMAL | 인앱 |
| 시그널 발생 | NORMAL | 인앱(+ 사용자 설정 시 이메일) |
| 모드 전환 | HIGH | 인앱 + 이메일 |
| CREON 연결 이상 | HIGH | 인앱 + 이메일 |

## 9. 사용자별 매매 모드 권한

| 사용자 등급 | SIM | LIVE | 비고 |
|---|:---:|:---:|---|
| ROLE_TRADER (일반) | O | X | 시뮬 전용 |
| ROLE_TRADER_PRO | O | O | 실거래 가능 |
| ROLE_OPERATOR | O | X | 운영 전용(매매 불가) |
| ROLE_ADMIN | O | O | 모든 권한 |

승급 절차: 시뮬 30건 + KYC 추가검증(향후 확장) → ROLE_TRADER_PRO 자동 승급 후보.

## 10. 백테스트 / 모의투자 정책 정합성

- 백테스트의 슬리피지/수수료/체결 가정은 시뮬레이션과 동일 기본값 사용.
- 백테스트 결과를 실거래 전략으로 등록 시, 시뮬 모드에서 최소 5영업일 운영 후 LIVE 승인 권장(소프트 가이드, 강제 아님).

---

## 11. 약관 / 책임 면책

- 모든 사용자는 가입 시 일반 약관에 동의한다.
- LIVE 모드 첫 활성화 시 별도 **자동매매 리스크 고지** 약관에 동의해야 한다.
- 약관 버전 변경 시 LIVE 사용자는 다음 LIVE 진입 시 재동의 필요.
- 시스템은 사용자 손실에 대해 자동매매 정책 범위 내에서 정확히 동작했음을 입증할 책임을 진다(감사 로그).

---

## 12. 정책 변경 절차

1. 정책 변경 제안서 작성(Planner).
2. PM 검토 + 승인.
3. 본 문서 버전 업데이트(v1.x).
4. 영향받는 컴포넌트(Risk Guard, Order Service, Frontend) 변경.
5. QA 정책 회귀 테스트 → 운영 반영.
