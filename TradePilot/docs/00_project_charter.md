## 1. 프로젝트 비전

한국 주식시장(KOSPI / KOSDAQ)을 대상으로, 업종·차트·기술적 지표 분석을 통한 추천주 도출과 매매 타이밍 알림, 그리고 자동매매·수익률 추적까지 **단일 플랫폼에서 일괄 제공**하는 개인 투자자용 자동매매 시스템을 구축한다.

## 2. 프로젝트 목표

### 2.1 비즈니스 목표
- 개인 투자자의 매매 의사결정 시간을 70% 이상 단축
- 감정 개입을 배제한 룰 기반·AI 기반 자동매매 환경 제공
- 시뮬레이션 모드로 사전 검증 후 실거래 전환이 가능한 안전 장치 확보

### 2.2 기술 목표
- 크레온 API(CREON Plus, Windows COM) 안정 연동
- 분석 → 신호 → 주문 → 체결 → 수익률 추적의 End-to-End 파이프라인 자동화
- 시뮬레이션 ↔ 실거래 토글 구조로 단일 코드베이스 유지
- LSTM 기반 단기(1~5일) 가격 예측 모델 운영화(MLOps 최소 수준)

## 3. 프로젝트 범위 (Scope)

### 3.1 포함 범위 (In-Scope)

| 영역 | 세부 내용 |
|---|---|
| 시장 데이터 | 실시간 시세, 일/분봉 데이터 수집·적재, 업종 지수 |
| 기술적 분석 | MA(5/20/60/120), RSI, MACD, 볼린저 밴드 |
| 거래량/모멘텀 | OBV, VWAP, Stochastic |
| 업종 분석 | 섹터 강도, 업종 로테이션, 상대강도(RS) |
| AI/ML 예측 | LSTM 기반 1~5일 단기 가격 예측, 신뢰도 점수 |
| 추천 엔진 | 멀티 인디케이터 점수 합산형 추천주 산출 |
| 매매 엔진 | 시뮬레이션(페이퍼) ↔ 실거래 토글, 시장가/지정가 주문 |
| 알림 | 매수/매도 시그널 알림(웹 푸시, 이메일) |
| 포트폴리오 | 보유종목, 평가손익, 일·월·누적 수익률, MDD |
| 백테스팅 | 과거 데이터 기반 전략 검증, 성과 리포트 |
| 사용자 | 개인 로그인, 전략 프로파일, 알림 설정 |
| 운영 | 일일 리포트, 장애 알림, 거래 로그 감사추적 |

### 3.2 제외 범위 (Out-of-Scope)

- 해외 주식(미국·일본 등) 매매
- 파생상품(선물·옵션·ETN) 거래
- 다중 증권사 동시 연동 (1차는 크레온 단일)
- 모바일 네이티브 앱 (반응형 웹으로 대응)
- 신용·미수 거래, 공매도
- 다중 사용자 동시 거래(B2C SaaS) - 1차는 단일 사용자 또는 소수 운영
- 세무 신고 자동화

## 4. 핵심 결정 사항 (Key Decisions)

| ID | 결정 사항 | 사유 |
|---|---|---|
| KD-01 | 증권사 API는 크레온(CREON Plus) 단일 | 개인 투자자 접근성, 무료 사용, 풍부한 API |
| KD-02 | 풀스택 1차 일괄 구축 (MVP 분리 미사용) | 매매 시스템 특성상 분석·실행·추적이 분리되면 운영 불가 |
| KD-03 | 시뮬레이션 ↔ 실거래 토글 단일 코드베이스 | 검증 일관성, 코드 중복 방지 |
| KD-04 | 크레온 게이트웨이는 Windows 별도 호스트로 분리 | Linux 기반 FastAPI 본체와 분리, COM 종속성 격리 |
| KD-05 | LSTM은 보조 지표(가중치 30% 이내) | 단독 신호로 사용 시 과적합·시장 변동 위험 |
| KD-06 | DB는 PostgreSQL | 시계열·집계 쿼리 성능, JSONB 활용 |

## 6. 성공 기준 (Success Criteria)

### 6.1 기능적 성공 기준
- 시뮬레이션 모드에서 4주 무중단 매매 사이클 검증 완료
- 백테스팅 결과 대비 실시간 신호 오차율 5% 이내
- 주문 → 체결 → 포트폴리오 반영 지연 3초 이내
- LSTM 예측 모델 RMSE 검증셋 기준 사전 합의 임계값 이내

### 6.2 비기능적 성공 기준
- 장중(09:00~15:30) 가용성 99.5% 이상
- 크레온 API 호출 제한(초당 15건) 위반 0건
- 실거래 사고(의도치 않은 주문) 0건
- QA 회귀 테스트 통과율 95% 이상

### 6.3 사용자 성공 기준
- 신규 사용자 온보딩 30분 이내 완료
- 대시보드 첫 화면 로딩 2초 이내

## 7. 주요 제약 사항 (Constraints)

- **기술 제약**: 크레온 API는 Windows COM 종속, 32-bit 프로세스 필수
- **시간 제약**: 한국 거래소 정규장(09:00~15:30) 외 시간엔 실거래 불가
- **법규 제약**: 자본시장법상 투자자문업·일임업 라이선스 미보유 → 개인 사용 한정, 외부 서비스 금지
- **예산 제약**: 1차 구축은 인프라 비용 최소화 (Docker Compose 기반 단일 서버)

## 8. 가정 (Assumptions)

- 사용자는 본인 명의 크레온 계좌를 보유
- 사용자 PC 또는 전용 Windows 호스트에서 크레온 Plus 상시 구동 가능
- 1차 운영 사용자는 5명 이내 (스케일 아웃 미고려)
- 시장 데이터는 크레온 API 한정 (별도 유료 데이터 벤더 미사용)
