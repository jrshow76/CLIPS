## 1. 릴리즈 전략 개요

TradePilot은 실거래 사고 방지를 최우선으로 하므로, **점진적 게이트 방식**으로 릴리즈한다. 시뮬레이션 검증 → 소액 실거래 파일럿 → 정식 릴리즈 순으로 단계별 게이트를 통과해야 다음 단계로 넘어간다.

```
Alpha (시뮬레이션 전용)
    ↓  QA 회귀 100% + 시뮬레이션 4주 검증
Beta (실거래 게이트 + 소액 한도)
    ↓  Beta 2주 무사고 + 일일 한도 도달 ≤ 1회/주
v1.0 (정식 릴리즈)
```

## 3. 환경 분리

| 환경 | 목적 | URL/호스트 | DB | 크레온 게이트웨이 | 데이터 |
|---|---|---|---|---|---|
| **Dev** | 개발자 로컬 | localhost | 로컬 PostgreSQL | 모의(Mock) 게이트웨이 | 샘플 데이터 |
| **QA / Staging** | QA·자동화 테스트 | staging.internal | 별도 PostgreSQL | 모의 + 시뮬 크레온 | 실시세 미러 (지연) |
| **Production** | 실사용 | prod.internal | 운영 PostgreSQL (HA) | 실 크레온 Plus (Windows 호스트 이중화) | 실시간 시세 |

### 3.1 환경별 설정

| 설정 | Dev | QA | Production |
|---|---|---|---|
| 매매 모드 기본값 | 시뮬레이션 | 시뮬레이션 | 시뮬레이션 |
| 실거래 토글 허용 | 차단 | 차단 | 허용 (이중확인 후) |
| 일일 한도 | 무제한 | 모의 | 사용자 설정 |
| 로그 레벨 | DEBUG | INFO | WARN + AUDIT |
| 모니터링 | 없음 | 기본 | 전체 + 알림 |

### 3.2 환경 승격 규칙

- Dev → QA: 개발자 PR 머지 시 자동 배포
- QA → Production: QA 완료 보고서 + PM 수동 승인 필수
- 운영 환경은 **수동 배포만 허용** (자동 배포 금지)

## 5. 배포 절차 (Deployment Procedure)

### 5.1 배포 시간대

| 환경 | 허용 시간 | 비고 |
|---|---|---|
| Dev | 24/7 | 자유 |
| QA | 평일 09:00~18:00 | QA와 사전 협의 |
| Production | **장 마감 후 (16:00 이후) 또는 주말** | 장중 배포 금지 |

> **장중 배포 절대 금지**: 정규장(09:00~15:30) 진행 중 운영 배포는 PM·DevLead 합의 + 긴급 사유 명시 시에만 예외 허용.

### 5.2 배포 단계

1. **배포 전 (D-1)**
   - 릴리즈 브랜치 생성, 태그 부여 (v1.0.0)
   - 배포 체크리스트 PM·DevLead·QA 사인오프
   - 사용자 사전 공지 (메일·메신저)

2. **배포 당일 (D-day, 16:30 이후)**
   - DB 백업 수행 (DBA)
   - DB 마이그레이션 (DBA)
   - 백엔드 컨테이너 롤링 배포 (DevLead)
   - 크레온 게이트웨이 재시작 (BackendSenior)
   - 프론트 정적 빌드 배포
   - 헬스체크 통과 확인

3. **배포 후 (D+0 ~ D+1)**
   - 시뮬레이션 모드 1시간 모니터링 (장 외 시간)
   - 익일 장 시작(09:00) 시 실시간 모니터링 대기 (DevLead·BackendSenior·PM)
   - 익일 장 마감 후 사후 보고서 작성

### 5.3 롤백 절차

- 트리거: 헬스체크 실패, 매매 사고 감지, Critical 결함 발견
- 절차:
  1. Kill Switch 발동 (모든 매매 중단)
  2. 직전 안정 버전 컨테이너 이미지로 롤백 (5분 이내)
  3. DB 마이그레이션 롤백 스크립트 적용 (DBA)
  4. 헬스체크 후 시뮬레이션 모드로 재가동
  5. PM 사고 보고서 작성

## 6. 버전 관리 정책

- v1.0.0: 정식 릴리즈
- v1.0.x: 패치 (버그 수정)
- v1.x.0: 마이너 (기능 추가, 하위 호환)

- 매매 로직 변경 시에는 Minor 이상 버전 상향 의무화
