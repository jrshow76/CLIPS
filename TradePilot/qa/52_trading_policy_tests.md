## 2. Kill Switch 5초 SLA

### 2.1 시나리오
| 케이스 ID | 트리거 | 사전 상태 | 기대 결과 |
|---|---|---|---|
| TP-KILL-001 | 사용자 수동 (UI 버튼) | LIVE, 미체결 3건 | 5초 내 자동매매 OFF + 미체결 0건 + SIM 강제 |
| TP-KILL-002 | 자동 (헬스체크 3회 실패) | LIVE 모드 | LIVE→SIM 강제 + 운영자 알림 |
| TP-KILL-003 | 자동 (일일 손실 -3% 도달) | LIVE 모드 | 자동매매 OFF + 보유 청산 옵션 실행 |
| TP-KILL-004 | 운영자 명령 (관리자 API) | 다중 사용자 LIVE | 전 사용자 LIVE→SIM, 진행 주문 취소 |
| TP-KILL-005 | 시장 이벤트 (서킷브레이커) | 시장 단위 | 모든 사용자 매매 정지 + 알림 |

### 2.2 SLA 검증 단계
1. **0초**: 신규 주문 큐 차단 플래그 ON (`order_router.blocked=true`).
2. **5초 이내**: 미체결 주문 일괄 취소 호출 → 결과 집계.
3. **10초 이내**: 매매 엔진 일시정지 + 인앱 알림.
4. **30초 이내**: PM/DevLead 알림(SMS/이메일).
5. **이후**: `audit_log` 1건 기록(트리거 주체, 처리 건수).

### 2.3 부분 실패 케이스
- 케이스 ID `TP-KILL-006`: 미체결 5건 중 1건 취소 실패.
- 기대: 502 `E0015`, 미처리 주문 ID 리스트 details 포함, audit_log 부분 실패 기록.

### 2.4 자동화 위치
- 백엔드: `backend/tests/qa/test_kill_switch.py`
- E2E: `qa/e2e/tests/auto-trading.spec.ts::kill_switch`
