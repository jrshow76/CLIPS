## 0. 공통(모든 릴리즈)

## 2. Beta 릴리즈 체크리스트 (목표: 2026-06-26)

> 실거래 + 강제 소액 한도(일일 100만원). QA 회귀 100%. 안전망 검증 강화.

### 2.1 Alpha 결함 수정
- [ ] Alpha 단계 발견 Critical/Major 결함 0건 (Jira 필터 검증)
- [ ] 회귀 자동화에 신규 케이스 등록 완료

### 2.2 시뮬레이션 4주 누적 검증
- [ ] 4주(20영업일) 시뮬 사이클 보고서 합격
- [ ] 누적 슬리피지/수수료 정합성 검증 (DBA SELECT 확인)
- [ ] LSTM 모델 정확도 모니터링 4회 이상 그린

### 2.3 Risk Guard 동작 검증 (P0 회귀)
- [ ] TP-LIMIT-001~008 통과 (한도)
- [ ] TP-LIMIT-010~013 통과 (손절/익절)
- [ ] TP-LOSS-001~005 통과 (일일 손실 한도)
- [ ] TP-FORCE-001~013 통과 (강제 청산)
- [ ] **Beta 소액 한도 하드 캡 100만원 적용 확인** (코드 + DB)
- [ ] TP-FORCE-SIM-001~004 통과 (자동 LIVE→SIM)

### 2.4 Kill Switch 동작 검증 (P0)
- [ ] TP-KILL-001 사용자 수동 5초 SLA 통과
- [ ] TP-KILL-002 자동 트리거(헬스체크 3회 실패) 통과
- [ ] TP-KILL-003 일일 손실 한도 자동 발동 통과
- [ ] TP-KILL-006 부분 실패 `E0015` 통과
- [ ] 30초 이내 PM/DevLead 알림 발송 확인

### 2.5 LIVE 전환 7단계 게이트 (P0)

- [ ] E2E `live-mode-modal.spec.ts` 통과
  - [ ] 비밀번호 입력 강제
  - [ ] 동의 문구 정확 일치 검증
  - [ ] 동의 문구 오타 시 버튼 비활성
- [ ] 약관 버전 변경 시 재동의 강제 확인

### 2.6 시장 이벤트 가드
- [ ] 서킷브레이커·VI 발동 시 매매 중단 시뮬레이션 통과

## 3. v1.0 정식 릴리즈 체크리스트 (목표: 2026-07-06)

> Beta 2주 무사고 + QA 회귀 100% + 운영 매뉴얼 완료.

### 3.1 Beta 운영 결과
- [ ] Beta 2주(10영업일) 사고 0건 (Kill Switch 발동 0회)
- [ ] 일일 한도 도달 ≤ 1회/주
- [ ] Critical/High 결함 잔존 0건
- [ ] 회귀 자동화 100% 그린(Beta 결함 회귀 케이스 포함)

### 3.2 DBA HA / 백업
- [ ] PostgreSQL HA 구성 검증 (Primary-Standby Failover)
- [ ] 일일 백업 + 주간 풀백업 + 월간 아카이브
- [ ] 백업 복원 리허설 통과 (RTO 1시간 이내)
- [ ] 인덱스 점검 + 슬로우 쿼리 TOP 10 < 1초

### 3.3 운영 매뉴얼
- [ ] 일상 운영 매뉴얼 (장 시작 전 점검, 일일 리포트)
- [ ] 장애 대응 매뉴얼 (CREON 단절, DB 다운, Kill Switch 절차)
- [ ] 복구 절차 (롤백, DB 복원, 시뮬 모드 재가동)
- [ ] 운영 알림 채널(메신저/이메일) 매핑 표 완성

### 3.4 사용자 가이드 / 법무
- [ ] 사용자 온보딩 가이드 (HTML/PDF)
- [ ] **개인 사용 한정 + 투자 책임 사용자** 고지문 UI 진입 시 표시
- [ ] 약관 / 면책 / 리스크 고지 PDF 첨부
- [ ] LIVE 모드 첫 활성화 시 약관 동의 강제

### 3.5 SLA / 모니터링
- [ ] 장중 가용성 SLA 99.5% 측정 가능
- [ ] 장애 1차 대응 < 15분 알람 동작
- [ ] MTTR < 1시간 절차 검증
- [ ] 자동 일일 리포트 발행 (PM/사용자 메일)

### 3.6 데이터 정합성 (DBA 협업)
- [ ] 주문→체결 정합성 SELECT 검증 (orders join trades count 일치)
- [ ] 잔고 vs 거래 내역 합산 정합성 검증
- [ ] 백테스트 결과와 시뮬 결과 슬리피지·수수료 동일 가정

### 3.7 사인오프
- [ ] QA Lead 사인오프
- [ ] DevLead 사인오프
- [ ] PM 사인오프
- [ ] 스폰서 최종 승인 (서면)

## 4. 배포 당일 D-day 체크리스트

> 장 마감 후(16:00 이후) 진행. 장중 배포 금지.

### 4.1 배포 전 (16:00~16:30)
- [ ] 사용자 사전 공지 발송 완료
- [ ] 사용자 LIVE 모드 일시 SIM 강제 (TP-FORCE-SIM-004)
- [ ] DB 백업 수행 (DBA)
- [ ] 직전 안정 버전 이미지 태깅 보존

### 4.2 배포 (16:30~17:30)
- [ ] DB 마이그레이션 (DBA)
- [ ] 백엔드 컨테이너 롤링 배포 (DevLead)
- [ ] 크레온 게이트웨이 재시작 (BackendSenior)
- [ ] 프론트 정적 빌드 배포
- [ ] 헬스체크 200 확인

### 4.3 배포 후 (17:30~)
- [ ] 시뮬레이션 모드 1시간 모니터링
- [ ] 익일 09:00 시작 전 운영진 대기 (DevLead/BackendSenior/PM)
