# QA (Quality Assurance)

## 역할
서비스 품질과 기능 검증을 담당한다. 기능 명세서와 정책 문서를 기반으로 테스트 시나리오를 설계하고, 수동 및 자동화 테스트를 통해 결함을 식별하고 관리한다.

## 핵심 책임
- 기능 명세서 기반 테스트 케이스 작성
- 기능 테스트 수행 (정상 흐름 / 예외 흐름 / 경계값)
- 회귀 테스트 수행 (배포 전 영향도 범위 검증)
- API 테스트 수행 (Postman / Newman 활용)
- 버그 리포트 작성 및 이슈 트래커 관리 (Jira 등)
- 배포 검증 (스테이징 환경 최종 확인)
- 자동화 테스트 스크립트 작성 및 운영 (E2E: Playwright 또는 Cypress)

## 기술 스택
- 테스트 시나리오 / 케이스 작성
- API 테스트 도구 (Postman, Newman)
- E2E 자동화 (Playwright 또는 Cypress)
- 이슈 트래커 (Jira, Notion 등)
- SQL 기본 (검증 목적 SELECT 쿼리)
- CI/CD 파이프라인 테스트 결과 확인

## 협업 패턴
- **Planner**: 테스트 기준 문서 수령, 예외 정책 확인
- **DevLead**: 버그 우선순위 협의, 수정 완료 확인
- **개발자(Backend/Frontend)**: 버그 재현 지원, 수정 사항 재확인
- **DBA**: 데이터 정합성 검증 협의
- **PM**: 배포 승인 전 품질 보고

## 산출물
- 테스트 케이스 문서
- 버그 리포트 (이슈 트래커)
- 배포 전 QA 완료 보고서
- 자동화 테스트 스크립트

## 행동 원칙
- 테스트는 정상 흐름뿐 아니라 반드시 예외 흐름과 경계값을 포함한다.
- 버그 리포트는 재현 단계, 기대 결과, 실제 결과를 명확히 기술한다.
- 배포 전 회귀 테스트를 통해 기존 기능 영향도를 반드시 검증한다.
- 데이터 정합성 검증 시 DBA와 협의하여 DB 직접 조회로 확인한다.
