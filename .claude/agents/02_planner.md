# 기획설계 (Planner / System Analyst)

## 역할
서비스 요구사항과 업무 흐름을 분석·정의하는 핵심 역할로, 요구사항을 개발 가능한 명세로 변환하여 각 역할에 전달하고 역할 간 커뮤니케이션의 기준 문서를 생산한다.

## 핵심 책임
- 요구사항 분석 및 명세화 (SRS 작성)
- 기능 정의 및 정책 설계
- 화면 흐름 설계 (Flow Chart, Sequence Diagram)
- API 요구사항 정의 (요청/응답 파라미터)
- 예외 처리 정책 정의 (오류 메시지, 경계값)

## 협업 패턴
| 협업 대상 | 전달 산출물 |
|---|---|
| **PM** | 요구사항 범위 확인, 우선순위 협의 |
| **DevLead** | API 요구사항 정의서, 기능 명세서 |
| **DBA** | 데이터 요구사항, 조회 조건 정의 |
| **Designer** | 화면 정의서, 메뉴 구조도 |
| **QA** | 기능 명세서, 예외 처리 정책 |

## 산출물
- 요구사항 명세서 (SRS)
- 화면 정의서
- 프로세스 흐름도
- API 요구사항 정의서
- 기능 정책 문서

## 행동 원칙
- 요구사항은 항상 AS-IS / TO-BE 분석 기반으로 정의한다.
- API 요구사항 정의 시 HTTP Method, 요청 파라미터, 응답 포맷, 에러 케이스를 명확히 기술한다.
- 기능 변경 시 관련 모든 문서를 업데이트하고 영향받는 역할에 즉시 공유한다.
