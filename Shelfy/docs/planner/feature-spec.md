## 1. 사용자 인증

### 1.1 회원가입

| 항목 | 내용 |
|---|---|
| 기능 ID | AUTH-001 |
| 기능명 | 회원가입 |
| 설명 | 신규 사용자가 이메일, 비밀번호, 닉네임으로 계정을 생성한다. |
| 접근 권한 | 비로그인 사용자 |

**입력 항목**

| 필드명 | 타입 | 필수 | 유효성 규칙 |
|---|---|---|---|
| email | String | Y | 이메일 형식, 최대 255자, 중복 불가 |
| password | String | Y | 8~20자, 영문+숫자+특수문자 각 1자 이상 포함 |
| passwordConfirm | String | Y | password 필드와 동일해야 함 |
| nickname | String | Y | 2~20자, 한글/영문/숫자/언더스코어만 허용, 중복 불가 |
| agreeTerms | Boolean | Y | true여야 가입 가능 |
| agreePrivacy | Boolean | Y | true여야 가입 가능 |
| agreeMarketing | Boolean | N | 선택 동의 |

**출력 (성공)**

- HTTP 201 Created
- 응답 바디: 생성된 사용자 기본 정보 (userId, email, nickname)
- 이메일 인증 메일 발송

**예외 처리**

| 예외 코드 | 상황 | 응답 메시지 |
|---|---|---|
| AUTH-E001 | 이메일 중복 | "이미 사용 중인 이메일입니다." |
| AUTH-E002 | 닉네임 중복 | "이미 사용 중인 닉네임입니다." |
| AUTH-E003 | 비밀번호 불일치 | "비밀번호가 일치하지 않습니다." |
| AUTH-E004 | 이메일 형식 오류 | "올바른 이메일 형식을 입력하세요." |
| AUTH-E005 | 비밀번호 규칙 위반 | "비밀번호는 8~20자, 영문·숫자·특수문자를 포함해야 합니다." |
| AUTH-E006 | 필수 동의 누락 | "필수 약관에 동의해야 합니다." |

### 1.2 이메일 인증

| 항목 | 내용 |
|---|---|
| 기능 ID | AUTH-002 |
| 기능명 | 이메일 인증 |
| 설명 | 회원가입 후 발송된 인증 링크를 클릭하여 이메일 인증을 완료한다. |

**정책**

- 인증 링크 유효 시간: 24시간
- 미인증 상태에서 로그인은 가능하나, 셀러 기능(상품 등록) 사용 불가
- 인증 링크 만료 시 재발송 요청 가능

**예외 처리**

| 예외 코드 | 상황 | 응답 메시지 |
|---|---|---|
| AUTH-E010 | 인증 토큰 만료 | "인증 링크가 만료되었습니다. 재발송을 요청하세요." |
| AUTH-E011 | 유효하지 않은 토큰 | "유효하지 않은 인증 링크입니다." |
| AUTH-E012 | 이미 인증 완료 | "이미 이메일 인증이 완료된 계정입니다." |

### 1.3 로그인

| 항목 | 내용 |
|---|---|
| 기능 ID | AUTH-003 |
| 기능명 | 로그인 |
| 설명 | 등록된 이메일과 비밀번호로 인증 후 Access Token / Refresh Token을 발급한다. |
| 접근 권한 | 비로그인 사용자 |

**입력 항목**

| 필드명 | 타입 | 필수 | 유효성 규칙 |
|---|---|---|---|
| email | String | Y | 이메일 형식 |
| password | String | Y | 비어 있지 않아야 함 |

**출력 (성공)**

- HTTP 200 OK
- 응답 바디: `accessToken` (JWT, 유효 1시간), `refreshToken` (유효 14일, HttpOnly 쿠키)
- `refreshToken`은 쿠키에만 저장, 클라이언트 JS 접근 불가

**정책**

- 5회 연속 로그인 실패 시 30분 계정 잠금
- 잠금 해제는 30분 경과 또는 비밀번호 재설정으로 가능

**예외 처리**

| 예외 코드 | 상황 | 응답 메시지 |
|---|---|---|
| AUTH-E020 | 이메일 미존재 또는 비밀번호 불일치 | "이메일 또는 비밀번호를 확인하세요." |
| AUTH-E021 | 계정 잠금 | "로그인 5회 실패로 계정이 잠금되었습니다. 30분 후 재시도하거나 비밀번호를 재설정하세요." |
| AUTH-E022 | 탈퇴된 계정 | "탈퇴된 계정입니다." |

## 2. 셀러 - 상품 관리

### 2.1 상품 등록

| 항목 | 내용 |
|---|---|
| 기능 ID | ITEM-001 |
| 기능명 | 상품 등록 |
| 설명 | 셀러가 판매할 물건을 선반에 등록한다. |
| 접근 권한 | 이메일 인증 완료한 로그인 사용자 |

**입력 항목**

| 필드명 | 타입 | 필수 | 유효성 규칙 |
|---|---|---|---|
| title | String | Y | 2~100자 |
| description | String | Y | 10~5000자 |
| category | String | Y | 카테고리 코드 (Enum) |
| saleType | String | Y | `PURCHASE` / `SUBSCRIBE` / `BOTH` |
| price | Integer | 조건부 | saleType이 PURCHASE 또는 BOTH인 경우 필수. 100원 이상, 10,000,000원 이하 |
| subscriptionPlans | Array | 조건부 | saleType이 SUBSCRIBE 또는 BOTH인 경우 1개 이상 필수 |
| images | File[] | Y | 최소 1장, 최대 10장, 각 최대 10MB, JPG/PNG/WEBP |
| thumbnailIndex | Integer | N | 대표 이미지 인덱스 (기본값: 0) |
| tags | String[] | N | 최대 10개, 각 최대 20자 |
| status | String | N | `DRAFT` / `PUBLISHED` (기본값: `DRAFT`) |

**구독 플랜 (subscriptionPlans 배열 항목)**

| 필드명 | 타입 | 필수 | 유효성 규칙 |
|---|---|---|---|
| planName | String | Y | 2~50자 (예: "Basic", "Premium") |
| period | String | Y | `MONTHLY` / `QUARTERLY` / `YEARLY` |
| planPrice | Integer | Y | 100원 이상 |
| description | String | N | 최대 500자 |

**출력 (성공)**

- HTTP 201 Created
- 응답 바디: 생성된 상품 ID (`itemId`)

**예외 처리**

| 예외 코드 | 상황 | 응답 메시지 |
|---|---|---|
| ITEM-E001 | 이메일 미인증 | "이메일 인증 후 상품을 등록할 수 있습니다." |
| ITEM-E002 | 이미지 형식 오류 | "JPG, PNG, WEBP 형식의 이미지만 업로드 가능합니다." |
| ITEM-E003 | 이미지 용량 초과 | "이미지 1장당 최대 10MB까지 업로드 가능합니다." |
| ITEM-E004 | 이미지 개수 초과 | "이미지는 최대 10장까지 등록 가능합니다." |
| ITEM-E005 | 가격 범위 초과 | "가격은 100원 이상 10,000,000원 이하여야 합니다." |
| ITEM-E006 | 카테고리 오류 | "유효하지 않은 카테고리입니다." |
| ITEM-E007 | 구독 플랜 누락 | "구독 상품은 최소 1개의 플랜을 설정해야 합니다." |

### 2.2 상품 수정

| 항목 | 내용 |
|---|---|
| 기능 ID | ITEM-002 |
| 기능명 | 상품 수정 |
| 설명 | 등록한 상품의 정보를 수정한다. |
| 접근 권한 | 해당 상품의 셀러 본인 |

**정책**

- 판매 중(`PUBLISHED`) 상태의 상품도 수정 가능
- 수정 시 기존 이미지 유지 또는 교체 가능 (이미지 ID 목록으로 관리)
- 구독자가 존재하는 구독 플랜의 `planPrice`는 변경 불가 (플랜 추가/비활성화는 가능)
- 상품 수정 이력은 별도로 기록하지 않음 (추후 확장 고려)

**예외 처리**

| 예외 코드 | 상황 | 응답 메시지 |
|---|---|---|
| ITEM-E020 | 권한 없음 | "해당 상품을 수정할 권한이 없습니다." |
| ITEM-E021 | 구독자 존재하는 플랜 가격 변경 시도 | "구독자가 있는 플랜의 가격은 변경할 수 없습니다." |
| ITEM-E022 | 존재하지 않는 상품 | "상품을 찾을 수 없습니다." |

### 2.3 상품 삭제

| 항목 | 내용 |

| 기능 ID | ITEM-003 |

| ITEM-E030 | 활성 구독자 존재 | "활성 구독자가 있는 상품은 삭제할 수 없습니다. 구독 종료 후 삭제하세요." |
| ITEM-E031 | 권한 없음 | "해당 상품을 삭제할 권한이 없습니다." |

## 3. 바이어 - 탐색 및 검색

### 3.1 상품 목록 탐색 (홈/탐색 피드)

|---|---|
| 기능 ID | BROWSE-001 |
| 기능명 | 상품 목록 탐색 |
| 설명 | PUBLISHED 상태의 상품을 카드 형식으로 표시한다. |
| 접근 권한 | 전체 공개 (비로그인 포함) |

## 7. 공통 정책

### 7.1 카테고리 코드

| 코드 | 명칭 |
|---|---|
| DIGITAL_CONTENT | 디지털 콘텐츠 |
| COURSE | 강의/교육 |
| TEMPLATE | 템플릿/자료 |
| PHOTO | 사진/그래픽 |
| MUSIC | 음악/사운드 |
| SOFTWARE | 소프트웨어/앱 |
| OTHER | 기타 |

### 7.2 공통 응답 포맷

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "timestamp": "2026-05-09T12:00:00Z"
}
```

**오류 응답**

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "AUTH-E001",
    "message": "이미 사용 중인 이메일입니다."
  },
  "timestamp": "2026-05-09T12:00:00Z"
}
```

### 7.3 인증 정책

- 인증이 필요한 API는 Authorization 헤더에 Bearer 토큰 필수
- 토큰 누락 또는 만료 시 HTTP 401 반환
- 권한 없는 리소스 접근 시 HTTP 403 반환

### 7.4 파일 업로드 정책

- 지원 형식: JPG, PNG, WEBP
- 이미지 저장: CDN(Object Storage) 연동
- 업로드 완료 시 CDN URL 반환

### 7.5 페이지네이션 공통 응답

```json
{
  "content": [ ... ],
  "page": 0,
  "size": 20,
  "totalElements": 150,
  "totalPages": 8,
  "first": true,
  "last": false
}
```
