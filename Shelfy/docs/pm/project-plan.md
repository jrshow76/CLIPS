## 1. 프로젝트 개요

### 1.1 목적

Shelfy는 사용자가 자신의 물건을 선반(Shelf)에 등록하고, 다른 사용자가 해당 선반을 둘러보며 아이템을 구매하거나 구독할 수 있는 C2C 기반 웹 애플리케이션이다. 개인의 소장품을 큐레이션 형태로 공개하여 공유 경제 생태계를 형성하는 것을 목표로 한다.

### 1.2 목표

- 1차 MVP 출시: 2026-08-29 (약 16주)
- 핵심 사용자 시나리오(선반 개설, 상품 등록, 구경, 구매, 구독) 구현 완료
- 안정적인 서비스 운영 기반(Docker, CI/CD) 확보

### 1.3 기술 스택

| 구분 | 기술 |
|---|---|
| Backend | Java, Spring Boot, Spring Batch, MyBatis / JPA |
| Frontend | React, Next.js, TypeScript, TanStack Query |
| Database | PostgreSQL |
| Infra | Docker, Docker Compose, GitHub Actions |
| API 테스트 | Postman, Newman |
| E2E 테스트 | Playwright |

## 3. 주요 기능 범위 (1차 MVP)

### 3.1 포함 범위 (In-Scope)

#### 사용자 관리
- 회원가입 / 로그인 / 로그아웃 (이메일 기반)
- 소셜 로그인 (Google OAuth)
- 프로필 설정 (닉네임, 프로필 사진, 소개글)
- 비밀번호 변경 / 회원 탈퇴

#### 선반(Shelf) 관리
- 선반 개설 (이름, 설명, 카테고리, 커버 이미지)
- 선반 수정 / 삭제 / 공개·비공개 전환
- 선반 목록 탐색 (카테고리 필터, 검색)
- 선반 상세 조회 (등록된 아이템 목록)

#### 아이템(Item) 관리
- 아이템 등록 (이름, 설명, 가격, 이미지 최대 5장, 카테고리)
- 아이템 수정 / 삭제 / 판매 상태 변경
- 아이템 상세 조회 (이미지 슬라이더, 판매자 정보)

#### 구매
- 아이템 구매 요청 (간이 결제 플로우 - 외부 PG 연동은 2차)
- 구매 내역 조회 (구매자/판매자 양측)
- 구매 상태 관리 (요청 → 확인 → 완료 / 취소)

#### 구독
- 선반 구독 / 구독 해제
- 구독한 선반의 신규 아이템 알림 (인앱 알림)
- 구독 선반 목록 조회

#### 탐색 및 검색
- 홈 피드 (인기 선반, 최신 아이템)
- 통합 검색 (선반명, 아이템명, 판매자명)
- 카테고리별 탐색

#### 알림
- 인앱 알림 (구독 선반 신규 아이템, 구매 상태 변경)
- 알림 읽음 처리 / 목록 조회

#### 관리자 기능 (최소)
- 회원 목록 조회 / 강제 탈퇴
- 신고된 아이템 검토 / 숨김 처리

### 3.2 제외 범위 (Out-of-Scope, 2차 이후)

- 외부 PG 실결제 연동 (카카오페이, 토스페이 등)
- 채팅 / 실시간 메시지 기능
- 경매 기능
- 모바일 앱 (iOS / Android)
- 다국어 지원
- 셀러 정산 자동화 (Spring Batch 2차)
- 추천 알고리즘 고도화
- 이메일 / SMS 알림

## 5. 우선순위 정의

### 5.1 기능 우선순위 (MoSCoW)

| 우선순위 | 기능 |
|---|---|
| Must Have | 회원가입/로그인, 선반 개설/조회, 아이템 등록/조회, 구독, 홈 피드 |
| Should Have | 구매 플로우, 통합 검색, 인앱 알림, 프로필 관리 |
| Could Have | 관리자 기능, 카테고리 탐색, 신고 기능 |
| Won't Have (1차) | 실결제 PG 연동, 채팅, 경매, 모바일 앱 |
