---

## 보고서 작성 규칙

1. **배포 전 반드시 작성**하여 PM에게 제출한다.
2. **미해결 Critical/High 버그가 0건**일 때만 배포 승인 요청이 가능하다.
3. **테스트 케이스 통과율 95% 이상**을 달성해야 배포를 권고한다.
4. PM은 이 보고서를 확인 후 배포를 승인한다.

## 보고서 제출 프로세스

1. QA 완료 보고서 작성 완료 → PM에게 이메일/Notion으로 전달
2. PM이 보고서 검토 (미해결 Critical/High 버그 없음 확인)
3. PM 최종 배포 승인 서명
4. DevLead 운영 배포 진행
5. QA 스모크 테스트 수행 (배포 직후)
6. PM 서비스 오픈 공표
